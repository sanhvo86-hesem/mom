ENTERPRISE SCOPE
- This pack is not evidence-only.
- It is the enterprise file-plan template for all departments, all roles, and all employees.
- Department shared records, team workbenches, named employee workbenches, restricted people dossiers, digital admin records, and controlled source roots are all included.

EMPLOYEE STORAGE MODEL
- Each employee has up to 3 storage loci:
  - official records in the correct department/library/job series
  - a named employee workbench under the employee's role in 05-Department-Records
  - a restricted people dossier in 08-People-Records when the content is HR-sensitive
- Do not create personal SSOT folders outside this structure.

PLACEHOLDER FOLDERS
- Folders wrapped in braces are templates and must be copied or renamed for real use.
- Examples:
  - {CustomerID}
  - {JobNum}-{PartNo}-REV-{Revision}
  - {EmployeeID}-{FullName}
  - {CandidateOrPersonID}-{Name}
  - {Population}
  - {RestrictedProcess}
  - {VisitType}
  - {VisitorName}
  - {PartNo}
  - REV-{Rev}

SYNC-SAFE PATH RULE
- This template is designed for SharePoint + OneDrive sync + Windows local worktree use.
- Keep the short deep-branch segments exactly as issued:
  - 90-Employee-WB
  - 03-Team-Promote
  - 04-Team-Offline
  - 04-My-Offline-Pack
  - 02-Pending-Starters
  - 06-Restricted-Shared-Ops
- Do not expand these back into longer sentences.
- In deep folders, file names must stay short and structured because the path already carries department, role, employee, year and record-series context.

PART MASTER RULE
- Part-REV-Master is customer-scoped.
- Use Part-REV-Master/{CustomerID}/{PartNo}/REV-{Rev}/.
- Do not store two different customer baselines under the same PartNo branch.

JOB DOSSIER RULE
- Job dossier templates are customer-scoped.
- Use 03-Job-Dossiers/{YYYY}/{CustomerID}/{JobNum}-{PartNo}-REV-{Revision}/.

IMPORTANT UPLOAD NOTE
- SharePoint browser upload may skip empty folders.
- Prefer OneDrive sync or File Explorer copy when you need the full empty tree to appear in SharePoint.
- If you upload by browser, put at least one real file into each folder branch that must exist.
