81Module Build Packet — 24 trường bắt buộc khi build module TARGET

Module Build Packet là tập hợp tất cả thông tin cần thiết để build, test, deploy và audit một module. Mỗi module PHẢI có một build packet trước khi submit QA gate.

CROSS-REF: Standard 36 Sections 8 và 9 Build packet là artifact machine-readable được Standard 36 yêu cầu cho mọi module mới hoặc thay đổi. Xem Standard 36 Section 8 cho module build packet và Standard 36 Section 9 cho template registry contract.
IMPLEMENTATION STATUS Đây là TARGET cho v4.6. Module Builder cần generate skeleton JSON, validate schema, kiểm tra template registry và block contracts trước khi cho module vào QA gate ở Section 56.

82Block Contract — 14 trường bắt buộc theo Standard 36 TARGET

Mỗi block type phải có contract machine-readable trước khi dùng trong production. Contract này nâng cấp catalog ở Section 37 và schema nền ở Section 29 bằng các field còn thiếu cho runtime, QA, accessibility, permission, responsive và print/export.

CROSS-REF: Standard 36 Sections 8, 9 và 12 Standard 36 định nghĩa đầy đủ build packet, template registry và block contract requirements. Xem Standard 36 Section 8, Section 9 và Section 12.
WARNING Đây là TARGET. Block contracts chưa bắt buộc cho modules hiện tại. Từ v4.7, Module Builder sẽ validate block contracts khi build. Các modules cũ có grace period 2 sprint để bổ sung.

Field Type Mô tả
blockTypestringTên block ổn định, ví dụ data-table.
versionsemverPhiên bản block contract; breaking change phải tăng major.
ownerstringTeam chịu trách nhiệm maintain, test và release block.
propsSchemaJSON SchemaSchema cho props input, required fields, enum và default values.
dataSourceSchemaJSON SchemaEndpoint, query, variables, refresh và cache model.
eventContractobjectEvents block emit và listen, payload shape và failure behavior.
stateContractobjectInternal state: loading, empty, error, readonly, disabled, dirty, saving, saved, conflict, permission-denied.
permissionContractobjectRequired permissions, visible/enabled/masked/denied behavior.
a11yContractobjectARIA roles, keyboard, focus management, live region và labels.
responsiveContractobjectBehavior tại mỗi breakpoint/container, min/max dimensions và overflow.
themeContractobjectTokens block sử dụng, contrast rules và high-contrast behavior.
printExportContractobjectPrint, PDF, CSV, XLSX hoặc not applicable.
testIdsobjectStable data-testid map cho UI tests.
qaEvidenceobjectTest results, screenshots, a11y audit và review evidence.
