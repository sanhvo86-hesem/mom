Patch Section 22Virtual Scroll Table Contract
VIRTUAL SCROLL Tables > 500 rows PHẢI dùng virtual scroll để giữ interaction ổn định. Implementation render viewport + 20 rows buffer trên/dưới, giữ row height cố định theo density và không làm mất keyboard navigation.
Aspect Spec ARIA / Keyboard Performance Fallback
Trigger Tables > 500 rows PHẢI dùng virtual scroll; dưới ngưỡng này có thể dùng pagination hoặc table thường. Table giữ role/grid semantics hiện có. Không render toàn bộ DOM nếu row count vượt ngưỡng. Nếu virtualization lỗi, fallback pagination có page size tối đa 100.
Render window Render viewport + 20 rows buffer trên/dưới; recycle row DOM khi scroll. aria-rowcount trên table, aria-rowindex trên mỗi row theo index thật trong dataset. Scroll target 60fps, row layer dùng will-change: transform khi đang scroll. Skeleton rows khi scroll nhanh qua buffer hoặc data page chưa tới.
Row height Fixed theo density: compact 32px, default 40px, comfortable 48px. Không cho nội dung động làm tăng row height; expandable detail phải nằm ngoài row virtualized chính. Layout tính toán bằng fixed height để tránh reflow liên tục. Text dài truncate hoặc wrap trong detail panel có chiều cao riêng.
Keyboard ArrowUp/ArrowDown vẫn di chuyển từng row; PageUp/PageDown nhảy 20 rows. Roving tabindex cập nhật theo row thật; focus không rơi khỏi viewport khi row DOM recycle. Scroll into view dùng transform, không ép synchronous layout trên mỗi keydown. Nếu screen reader mode không ổn định, cung cấp pagination accessible alternative.

const focusStack = []

function openModal(level, trigger) {
  focusStack.push({
    level,
    returnTo: document.activeElement || trigger
  })
  focusInitialElement(level)
}

function closeModal(level) {
  const current = focusStack.pop()
  const previous = focusStack[focusStack.length - 1]
  const target = previous?.returnTo || current?.returnTo
  if (target && document.contains(target)) target.focus()
}
