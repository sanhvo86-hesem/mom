AKiến trúc hệ thống

Template-centric architecture. Token cascade 5 lớp. Tab System mới 6 tabs. Template-Module binding bắt buộc.

Token Cascade — 5 lớp ghi đè
Lớp 1 — System CSS
hesem-design-tokens.css — 181 CSS variables mặc định. Nền tảng bất biến.
Lớp 2 — Admin Config
design-system-config.json — Admin tùy chỉnh brand, density, typography, colors cho tổ chức.
Lớp 3 — Template Override
Mỗi template có thể override tokens: font-size, spacing, colors, density riêng.
Lớp 4 — Zone Override
Mỗi zone trong template có thể override thêm: sidebar font nhỏ hơn, main spacing lớn hơn.
Lớp 5 — User Preference
localStorage hesem_user_appearance — dark mode, font size, density cá nhân.

BQuy chuẩn đồ họa

Bộ quy tắc bắt buộc cho toàn bộ đồ họa phần mềm. Tham khảo SAP Fiori, Material 3, Atlassian, IBM Carbon, Ant Design.

📐 Hệ thống lưới (Grid System)
Base-8 spacing scale: 4, 8, 12, 16, 20, 24, 32, 40, 48, 56, 64, 80px
Grid: 12-column responsive
Breakpoints: sm(640) · md(768) · lg(1024) · xl(1280) · 2xl(1536)
Gutter: 12px (sm) · 16px (md) · 20px (lg) · 24px (xl)
Page padding: 16px (sm) · 24px (md) · 32px (lg) · 48px (xl)
🔤 Typography Scale
8 cấp: xs(11px) · sm(13px) · base(14px) · md(16px) · lg(18px) · xl(20px) · 2xl(24px) · 3xl(32px)
Line-height: tight(1.25) · normal(1.5) · relaxed(1.75)
Weights: 400(normal) · 500(medium) · 600(semibold) · 700(bold)
Quy tắc: Hero → 3xl bold mono · KPI → 2xl bold mono · Title → xl bold · Section → md semibold · Body → base · Caption → xs · Table header → xs uppercase tracking-wider
🎨 Color Theory
Status badge pattern: bg-{color}/10 text-{color} border-{color}/30
Surface hierarchy: Page → Surface → Surface Alt → Header → Modal
Dark mode: Surface #1e293b, Text #f1f5f9, Border #334155
♿ Contrast & Accessibility (WCAG AA)
Contrast tối thiểu: 4.5:1 (text thường) · 3:1 (text ≥18px bold)
Focus ring: 3px, offset 0, rgba(21,101,192,0.12)
Không dùng color alone để truyền đạt thông tin — luôn kèm icon/text
Reduced motion: prefers-reduced-motion: reduce → tắt animation
Keyboard navigation: Tab order logic, visible focus ring, Escape to close
📏 Spacing Rules
Component internal: 8px (nhỏ), 12px (TB), 16px (lớn)
Section gap: 24px
Card padding: 16-20px
Table cell: 8-12px (vertical), 12px (horizontal)
Filter bar: 12px 16px
🔲 Border & Radius
Width: 1px default, 2px focus/active
Style: solid default, dashed placeholder
Radius: sm(4px) · md(6px) · lg(8px) · xl(12px) · 2xl(16px) · full(9999px)
Controls → md · Cards → lg · Modals → xl · Badges → full
💫 Shadow & Motion
Shadow: xs · sm · md · lg · xl
Cards → sm · Dropdowns → md · Modals → lg
Motion: fast(100ms) · normal(150ms) · slow(250ms) · spring(300ms)
Easing: ease-out (enter), ease-in (exit)
🖼 Icon Standards
3 kích thước: sm(14px) · md(16px) · lg(20px)
Leading icon (kèm text): scale 1.12x, edge-trim 2px
Icon-only: scale 1.4x, inset 2px
Style: outline, stroke-width 2px
Icon-to-text gap: 6px
📊 Data Display Rules
IDs/codes: font-mono, text-primary (NCR-2026-1204)
Numbers: font-mono, right-aligned
Currency: format locale, right-aligned
Dates: dd/MM/yyyy hoặc relative ("2 giờ trước")
Status: Badge text-[10px] uppercase semantic colors
Percentage: progress bar + số

18Smart Scroll UX

Scroll contract của HESEM MOM đi theo nguyên tắc “chỉ data di chuyển”. Layout shell phải ổn định để người dùng luôn còn header, KPI, filter và action trong tầm nhìn.

Rule 1: Chỉ data scrollHeader, KPI, filter, tabs, footer giữ sticky. Chỉ main và sidebar được scroll và chỉ cuộn nội dung.
Rule 2: Overlay scrollbarScrollbar mặc định ẩn, hiện mảnh khi hover, không đẩy nội dung sang trái/phải.
Rule 3: Rê chuột ra để scroll trangKhi tổng layout dài hơn viewport, rê ra ngoài vùng data để scroll shell; rê vào data để scroll zone.
Rule 4: Search và filter luôn thấyKhông được để thanh lọc biến mất khi người dùng đang duyệt bảng hoặc kanban.
