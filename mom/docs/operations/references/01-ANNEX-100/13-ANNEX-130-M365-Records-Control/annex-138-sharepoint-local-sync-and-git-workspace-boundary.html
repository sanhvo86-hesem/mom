📎 TÀI LIỆU NÀY ĐÃ ĐƯỢC HỢP NHẤT VÀO ANNEX-136 §4 (RANH GIỚI SYNC SP↔GIT↔SERVER) — 2026-05-16
ANNEX-138 đang trên đường gộp vào ANNEX-136 §4 "Mô hình local sync bắt buộc" để giảm phân mảnh. Hiện tại còn giữ làm tham chiếu phụ; mọi nội dung mới về local sync model phải đưa vào ANNEX-136 §4, không thêm vào file này nữa.
Khi ANNEX-136 §4 đã chứa đủ nội dung tương đương (model worktree-in-OneDrive, path Windows chuẩn, recovery khi thay máy), ANNEX-138 sẽ chuyển sang trạng thái OBSOLETE.
ÁP DỤNG KHI
Dùng khi thiết kế endpoint local cho controlled source, xác định ai được phép sync source library, path nào được phép, path nào bị cấm và cách onboard hoặc phục hồi endpoint mới. ANNEX này là chuẩn về endpoint và workspace model, không phải WI thao tác click-by-click.

1. Mục đích và phạm vi

ANNEX này khóa cấu trúc endpoint local cho controlled source của HESEM. Phạm vi gồm: worktree được SharePoint sync, git-dir tách rời, cơ chế đăng ký endpoint, giới hạn của OneDrive/SharePoint sync, cấm nhiều người cùng ghi trên một worktree và cách recovery khi thay máy hoặc hỏng sync.

Ranh giới tài liệu: WI-107 khóa thao tác hằng ngày; ANNEX-136 khóa ranh giới giữa source, Git, server và runtime; SOP-101 và SOP-104 khóa governance tổng thể.

3. Kiến trúc endpoint bắt buộc

Bố cục được phép

SharePoint source library -> OneDrive sync folder -> eqms.hesemeng.com (worktree only) Git metadata -> C:\HESEM-Git\eqms.hesemeng.com.git

Bố cục bị cấm

SharePoint source library -> eqms.hesemeng.com -> .git -> node_modules runtime cache -> secrets -> session data

4. Quy tắc endpoint registration

RuleYêu cầu
Named endpointMỗi máy được phép sync source library phải có EndpointID, owner, OS, vị trí và approver trong register nội bộ.
One endpoint, one primary writerMỗi worktree chỉ có một người chịu trách nhiệm chính. Không dùng shared desktop hoặc generic account làm máy biên tập chính.
No unregistered syncKhông cho người dùng tự bấm sync source library trên máy chưa được duyệt.
Periodic reviewHàng quý phải rà lại các endpoint đang được sync và loại bỏ máy cũ, máy nghỉ việc hoặc máy đã đổi owner.

5. Giới hạn kỹ thuật của SharePoint / OneDrive phải tuân thủ

  • Không dùng mapped network drive hoặc junction/symlink làm nơi sync.
  • Không dùng cùng một worktree cho nhiều người luân phiên trên cùng máy mà không có bàn giao rõ ràng.
  • Không xem SharePoint sync là công cụ merge cho Git internals.
  • Không đưa thư mục có biến động runtime cao vào vùng sync source.
Nguồn chính thức: Microsoft Support - Work with synced files in File Explorer; Microsoft Support - Restrictions and limitations in OneDrive and SharePoint.

6. Cấm multi-writer trên cùng worktree

Tình huốngQuy định
Nhiều người cùng sửa trong một local synced worktreeBị cấm. Mỗi người dùng phải có endpoint riêng hoặc làm việc qua Git branch/merge đúng chuẩn.
Dùng USB/external drive chứa worktreeBị cấm.
Để team sửa trực tiếp trên server thay cho local synced worktreeBị cấm, trừ hotfix khẩn có reverse-sync close-out.

7. Những gì không được sync vào source worktree

Nhóm dữ liệuXử lý
Runtime luồng công việc binaries, sessions, ratelimit, cachesGiữ ở server hoặc runtime path; không sync vào worktree.
Secrets, keys, server-only credentialsGiữ ở server-only protected path; không đặt trong source library.
Submission payload, approvals, operational record dataPhải route về records libraries hoặc runtime record area; không đưa vào source worktree.
.git internalsĐặt ở git-dir riêng ngoài vùng sync.

8. Onboarding endpoint mới

  1. Phê duyệt endpoint theo register.
  2. Sync QMS-Source-Control xuống máy.
  3. Tạo git-dir riêng ngoài vùng sync.
  4. Clone bằng --separate-git-dir hoặc restore worktree theo đúng chuẩn.
  5. Chạy kiểm tra boundary và smoke sync trước khi cho phép biên tập.

9. Recovery khi thay máy hoặc hỏng sync

  1. Dừng sync trên máy cũ.
  2. Xác nhận mọi commit đã được push lên Git remote.
  3. Thu hồi endpoint cũ khỏi register.
  4. Provision endpoint mới theo mục 8.
  5. Xác nhận local pull và SharePoint sync hoàn tất trước khi biên tập lại.

10. Không được go-live khi

  • Worktree còn chứa .git trong vùng sync
  • Không xác định được endpoint owner
  • Source library đang mở external sharing
  • Chưa chứng minh được vòng local sync → Git → server → local sync
