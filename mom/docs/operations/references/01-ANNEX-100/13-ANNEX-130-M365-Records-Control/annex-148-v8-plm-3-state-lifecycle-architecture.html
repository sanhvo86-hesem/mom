v8 PLM 3-State Lifecycle Architecture

BỐI CẢNH: Sau 7 vòng đập-xây-lại (v4→v8), v8 đạt PLM/MES industry standard cho contract CNC manufacturer scale 200-500 nhân viên. Quan trọng nhất: Part-Master (engineering vault) và Job-Dossier (execution evidence) là 2 trục ngang hàng — không nest. Mọi authored content theo 3-state lifecycle (Working → In-Review → Released) chuẩn PLM (Teamcenter/Windchill/Aras pattern). Tổng blueprint: 747 (v7) → 1180 directories (v8.1) due to 3-state expansion + new Finance + Incident-Investigation libraries.

2. PLM Pattern — Part-Master và Job-Dossier ngang hàng (case study research)

Industry standard (Teamcenter/Windchill/Aras Innovator/Epicor/IFS):
PLM systems separate Engineering data (Part Master = drawings/CAM/inspection/FAI baseline — frozen on rev release) from Manufacturing data (Work Order/Job = execution evidence — live during production, frozen on Job close).
  • Part Master is engineering source-of-truth; multiple Jobs reference it
  • Job-Dossier consumes Part Master via snapshot reference (modern-link), not file copy
  • Cross-reference qua metadata: Job-Dossier metadata trỏ Parts; Part-Master metadata trỏ Jobs-using
