ÁP DỤNG KHI
Dùng khi quản lý tài liệu QMS, mã nguồn portal, định nghĩa biểu mẫu, manifest phát hành, biên nhận triển khai và hồ sơ đồng bộ ngược trên M365 SharePoint, nhưng vẫn phải biên tập cục bộ, quản trị bằng Git và triển khai lên server theo kiểm soát phát hành. ANNEX này khóa ranh giới giữa nguồn được kiểm soát, lịch sử phiên bản Git, bản đang chạy trên server và hồ sơ hoặc bằng chứng vận hành.

1. Mục đích, phạm vi và kết quả bắt buộc

ANNEX này quy định kiến trúc bắt buộc để HESEM đặt bộ nguồn trên M365 SharePoint mà không làm mờ chuỗi kiểm soát: máy cục bộ được phê duyệt dùng để biên tập, Git giữ lịch sử phiên bản, server chỉ nhận bản đã được phát hành, còn dữ liệu runtime phải tách hẳn khỏi thư viện nguồn. Người đọc phải xác định rõ bốn điểm: nguồn nằm ở đâu, máy nào được phép đồng bộ, Git giữ vai trò gì và dữ liệu nào tuyệt đối không được quay ngược vào cây nguồn.

KẾT QUẢ BẮT BUỘC: bộ tài liệu nguồn chỉ có một luồng chuẩn SharePoint local sync worktree → Git remote → server deploy; mọi hotfix buộc sửa trên server phải quay về Git → SharePoint local sync; dữ liệu runtime, bí mật hệ thống, binary nháp của luồng công việc, session files, trạng thái rate-limit và dữ liệu giao dịch của biểu mẫu không được trộn vào QMS-Source-Control.
Ranh giới tài liệu: ANNEX-133 khóa site, library và path; ANNEX-134 khóa cơ chế khởi tạo, phân quyền và tự động hóa; ANNEX-131 khóa từ điển field và list; ANNEX-135 khóa file plan; WI-102 khóa thao tác tạo site, library và quyền; WI-107 khóa thao tác đồng bộ cục bộ, Git và server.

4. Mô hình local sync bắt buộc

Đường dẫn local được phép

Chỉ các máy đã được IT/QMS chỉ định mới được sync library QMS-Source-Control. Worktree local phải nằm dưới thư mục sync do OneDrive quản lý trong hồ sơ người dùng, ví dụ:

%UserProfile%\HESEM Engineering\HESEM-Digital - QMS-Source-Control\01-Controlled-Source\eqms.hesemeng.com\

Git metadata được phép

Thư mục .git không được nằm trong thư mục đang sync với SharePoint. Git metadata phải được đặt ở path tách rời ngoài vùng sync, ví dụ:

C:\HESEM-Git\eqms.hesemeng.com.git\

Lý do kỹ thuật bắt buộc

  • Microsoft cho phép đồng bộ thư viện SharePoint xuống máy qua OneDrive hoặc File Explorer, nhưng không hỗ trợ đồng bộ vào network drive, mapped drive hoặc qua symbolic link hay junction. Thiết kế local workspace phải bám đúng ranh giới này.
  • Git metadata thay đổi rất dày; đặt .git trong vùng đang đồng bộ sẽ làm tăng nguy cơ xung đột sync, file lock và lịch sử nhiễu không liên quan đến nội dung thực.
  • Mô hình git work-tree tách khỏi git-dir cho phép worktree nằm trong vùng SharePoint sync mà không kéo toàn bộ Git internals vào library đang đồng bộ.

6. Luồng promotion chuẩn và luồng reverse sync

Luồng chuẩn

Worktree local đang đồng bộ SharePoint chỉ được chỉnh trên admin endpoint đã đăng ký, sau đó commit và push lên Git remote, rồi server pull hoặc deploy. Sau khi triển khai xong, phải lưu deploy receipt tại 03-Server-Deploy-Receipts.

Luồng hotfix từ server

Chỉ dùng khi buộc phải sửa point-of-use trực tiếp trên server. Sau hotfix, thay đổi phải được commit và push về Git remote ngay; local admin phải kéo ngược về worktree đồng bộ; và hồ sơ reverse sync phải được đóng tại 04-Reverse-Sync-Intake.

Luồng form

Định nghĩa biểu mẫu, schema đã phát hành và source template đi theo luồng nguồn chuẩn. Form entry, approval, runtime archive và submission payload không được đi vào Git hoặc source library trên SharePoint.

Cơ chế phát hành lên VPS thực tế (eqms.hesemeng.com — host chạy PHP 8.5 + nginx + PostgreSQL + Redis + RabbitMQ):
Đường mặc định: push vào branch main trên Git remote → GitHub Actions chạy .github/workflows/deploy.yml → SSH vào VPS, gọi /var/www/eqms.hesemeng.com/tools/vps-setup/scripts/deploy.sh. Script này chụp ảnh dữ liệu runtime vào /var/www/data-private/.deploy-snapshots/<TS>/, chạy reset có kiểm soát, sau đó phục hồi runtime theo xác thực sha256; nếu mismatch sẽ dừng và báo lỗi to.
Đường thủ công (khi CI không khả dụng): ssh eqms 'sudo -n bash /var/www/eqms.hesemeng.com/tools/vps-setup/scripts/deploy.sh' — bắt buộc dùng script này, không gõ trực tiếp git reset --hard, git restore, git clean -fd trên VPS.
Hot-fix tại VPS (hiếm): chỉ chỉnh qua giao diện admin của portal đi qua DataSyncMutationService (ghi snapshot + audit_events). Không sửa trực tiếp mom/data/config/*.json bằng vi/sed/cat>. Sau hotfix, đóng hồ sơ tại QMS-Source-Control/04-Reverse-Sync-Intake/{YYYY}/ theo §6 luồng reverse sync.
Tuyệt đối cấm trên VPS: bất kỳ thao tác Git phá huỷ nào (git reset --hard, git restore, git clean -fd) trên /var/www/eqms.hesemeng.com — đã có tiền lệ mất dữ liệu người dùng users.json khi rollback bằng git reset --hard origin/main qua SSH (sự cố commit 4a8ffebd); chính sách hiện hành neo tại CLAUDE.md §VPS deployment chính sách.

8.1. Luồng sync chi tiết: Online form → SharePoint

Khi người dùng gửi form trực tuyến trên web portal, server phải xử lý tuần tự theo các bước sau:

BướcHành độngChi tiết kỹ thuật
1Người dùng gửi form trên web portalServer nhận request POST tại api.php với payload đã qua kiểm tra phía client.
2Server lưu JSON entry làm bản gốcGhi vào qms-data/online-forms/entries/{form_code}.json. Đây là bản gốc duy nhất trên server.
3Server tạo PDF snapshotTên file bắt buộc: FRM-{code}_V{ver}_{scope}_{YYYYMMDD}_{HHMM}-{UserID}.pdf.
4Gán Record-ID khi loại hồ sơ yêu cầuÁp dụng cho NCR, CAPA, FAI và các loại hồ sơ cần mã định danh. Server phải cấp từ atomic counter sử dụng LOCK_EX để tránh trùng mã.
5Đồng bộ dữ liệu cấu trúc lên SharePoint ListServer gọi Microsoft Graph API để tạo hoặc cập nhật item trong SharePoint List tương ứng. Dữ liệu cấu trúc này phục vụ bảng điều khiển, báo cáo và truy vấn metadata.
6Upload PDF lên SharePoint Document LibraryServer upload PDF snapshot vào document library phù hợp theo file plan. PDF là bằng chứng phục vụ audit và thời hạn lưu trữ.
7Ghi allocation logMọi Record-ID đã cấp phải được ghi vào qms-data/counters/_allocation_log.jsonl theo kiểu append-only, không sửa và không xóa.

8.2. Luồng dành cho Excel form (offline fill)

Khi hồ sơ yêu cầu điền Excel form ngoại tuyến, luồng bắt buộc như sau:

BướcHành độngGhi chú
1Người dùng xin Record-ID qua portalChọn chức năng "Xin mã hồ sơ mới" trên portal. Server cấp mã từ atomic counter (LOCK_EX) và ghi allocation log.
2Người dùng tải blank form từ portalForm phải được đóng dấu phiên bản bằng mã form, phiên bản và ngày phát hành để bảo đảm đúng revision đang hiệu lực.
3Người dùng điền Excel form tại máy cá nhânTệp phải được lưu theo đúng naming convention bắt buộc trong file plan.
4Người dùng upload lại qua portalPortal tiếp nhận file Excel như controlled capture carrier, kiểm tra issuance ledger, version stamp, metadata và hash trước khi tạo submission attempt. SharePoint chỉ nhận bản snapshot chỉ-đọc do server publish sau khi hoàn tất.

9. Điều kiện không được go-live

  • Chưa tạo library QMS-Source-Control hoặc chưa khóa quyền đúng theo mô hình group.
  • Worktree local vẫn còn thư mục .git nằm trong vùng SharePoint sync.
  • Chưa chứng minh được trọn một vòng local sync → Git push → server pull → deploy receipt → local pull.
  • Runtime noise vẫn còn bị stage hoặc commit trong quá trình sync thử.
  • Chưa có quy tắc tách riêng giữa online form source và dữ liệu giao dịch của online form.
  • Chưa có WI thao tác cho admin endpoint và server operator.
