ÁP DỤNG KHI
Dùng khi đưa toàn bộ bộ tài liệu QMS, source portal, form definitions, release manifests, deploy receipts và reverse-sync control lên M365 SharePoint nhưng vẫn phải đi qua Git và server production có kiểm soát. ANNEX-136 - M365 Sharepoint Git Server Source Sync Promotion and Runtime Boundary

1. Mục đích, phạm vi và kết quả bắt buộc

ANNEX này xác lập kiến trúc bắt buộc để bộ tài liệu HESEM nằm trên M365 SharePoint theo cách vẫn hỗ trợ local authoring, Git promotion, server deployment, reverse sync và online-form governance mà không làm lẫn source với runtime data. Kết quả bắt buộc là: mọi Người biết bộ source docs nằm ở đâu trên SharePoint, local nào được phép sync, Git là gì trong chuỗi kiểm soát, server giữ cái gì, và dữ liệu nào tuyệt đối không được quay vào Git hay SharePoint source library.

KẾT QUẢ BẮT BUỘC: bộ tài liệu nguồn có đúng một đường đi chuẩn SharePoint local sync worktree → Git remote → server deploy; mọi hotfix từ server phải quay lại Git → SharePoint local sync; runtime data, secret-bearing config, draft workflow binaries, session files, rate-limit state và form transaction data không được trộn vào controlled source library.
Ranh giới tài liệu: ANNEX-133 khóa site/library/path; ANNEX-134 khóa provisioning, permissions và automation; ANNEX-131 khóa field/list catalog; ANNEX-135 khóa file plan; WI-102 khóa tạo site/library/quyền; WI-107 khóa thao tác click-by-click cho local sync, Git và server.

4. Mô hình local sync bắt buộc

Local path được phép

Chỉ các máy được chỉ định của IT/QMS mới được sync library QMS-Source-Control. Worktree local phải nằm dưới thư mục sync do OneDrive quản lý trong hồ sơ Người dùng, ví dụ:

%UserProfile%\HESEM Engineering\HESEM-Digital - QMS-Source-Control\01-Controlled-Source\qms.hesem.com.vn\

Git metadata được phép

Thư mục .git không được nằm trong thư mục đang sync với SharePoint. Git metadata phải tách sang path ngoài vùng sync, ví dụ:

C:\HESEM-Git\qms.hesem.com.vn.git\

Lý do kỹ thuật bắt buộc

  • Microsoft hỗ trợ sync thư viện SharePoint xuống máy qua OneDrive/File Explorer, nhưng OneDrive không hỗ trợ sync vào network drive hoặc mapped drive và không hỗ trợ sync bằng symbolic link/junction. Đây là ranh giới phải tôn trọng khi thiết kế local workspace.
  • Git metadata biến động rất nhiều; để .git trong vùng đang sync sẽ làm tăng nguy cơ xung đột sync, file lock và nhiễu history không liên quan đến nội dung tài liệu.
  • Mô hình git work-tree tách khỏi git-dir cho phép local "nằm trong SharePoint" mà không kéo toàn bộ internals Git vào library sync.

6. Luồng promotion chuẩn và luồng reverse sync

Luồng chuẩn

SharePoint-synced worktree được sửa tại local admin endpoint, sau đó commit và push lên Git remote, rồi server pull/deploy. Sau deploy, deploy receipt được lưu tại 03-Server-Deploy-Receipts.

Luồng hotfix từ server

Chỉ dùng khi phải sửa point-of-use trực tiếp trên server. Sau hotfix phải commit/push về Git remote ngay, local admin phải pull về worktree sync, và reverse-sync record phải được đóng tại 04-Reverse-Sync-Intake.

Luồng form

Form definitions, released schemas và source templates đi theo luồng source chuẩn. Form entries, approvals, runtime archives và submission payload không đi vào Git/SharePoint source library.

8.1. Luồng sync chi tiết: Online form → SharePoint

Khi Người dùng submit form trực tuyến trên web portal, các bước sau phải được thực hiện tuần tự bởi server:

BướcHành độngChi tiết kỹ thuật
1User submit form trên web portalRequest POST đến api.php với payload form data đã validate phía client.
2Server lưu JSON entry (primary storage)Ghi vào qms-data/online-forms/entries/{form_code}.json. Đây là bản gốc duy nhất trên server.
3Server tạo PDF snapshotTên file bắt buộc: FRM-{code}_V{ver}_{phạm vi}_{YYYYMMDD}_{HHMM}-{UserID}.pdf
4Gán Record-ID (nếu cần)Áp dụng cho NCR, CAPA, FAI và các loại hồ sơ yêu cầu mã định danh. Server gán từ atomic counter sử dụng LOCK_EX để tránh trùng mã.
5Sync structured data lên SharePoint ListServer gọi Microsoft Graph API để tạo/cập nhật item trong SharePoint List tương ứng. Dữ liệu structured phục vụ dashboard và báo cáo.
6Upload PDF lên SharePoint Document LibraryServer upload file PDF snapshot vào Document Library phù hợp theo file plan. PDF là evidence cho audit và retention.
7Ghi allocation logMọi Record-ID đã cấp được log vào qms-data/counters/_allocation_log.jsonl (append-only, không sửa/xóa).

8.2. Luồng dành cho Excel form (offline fill)

Khi hồ sơ yêu cầu điền Excel form ngoại tuyến, luồng bắt buộc như sau:

BướcHành độngGhi chú
1User yêu cầu Record-ID qua portalChọn chức năng "Xin mã hồ sơ mới" trên portal. Server cấp mã từ atomic counter (LOCK_EX) và ghi allocation log.
2User tải blank form từ portalForm được version-stamped (mã form, phiên bản, ngày phát hành) để đảm bảo đúng revision đang hiệu lực.
3User điền Excel form tại máy cá nhânLưu file với đúng naming convention bắt buộc theo file plan.
4User upload lại vào portalPortal tiếp nhận file Excel như controlled capture carrier, kiểm tra issuance ledger, version stamp, metadata và hash trước khi tạo submission attempt. SharePoint chỉ nhận bản snapshot/read-only do server publish sau finalization.

9. Điều kiện không được go-live

  • Chưa tạo library QMS-Source-Control và chưa khóa quyền đúng group model.
  • Worktree local vẫn chứa thư mục .git trong vùng SharePoint sync.
  • Chưa chứng minh được một vòng đầy đủ local sync → Git push → server pull → deploy receipt → local pull.
  • Runtime noise vẫn bị stage/commit trong quá trình sync thử Nhiệm.
  • Chưa có rule route riêng cho online form source và online form transaction data.
  • Chưa có WI thao tác cho admin endpoint và server operator.
