ÁP DỤNG KHI
Dùng khi thiết lập hoặc vận hành đường đi bắt buộc cho nguồn của bộ tài liệu HESEM: SharePoint local synced làm việc copy, Git version history, server triển khai, reverse sync và boundary với môi trường chạy data. Cách Mọi cũ "ShareFile" trong bộ tài liệu legacy được hiểu là SharePoint-synced local làm việc copy.

1. Kết quả phải đạt

  • Controlled nguồn nằm trong QMS-Source-Control/01-Controlled-Source/qms.hesem.com.vn/ trên SharePoint.
  • Local làm việc copy được sync bởi OneDrive/SharePoint, nhưng thư mục .git nằm ngoài vùng sync.
  • Mọi thay đổi nguồn đi theo đường SharePoint local sync → Git → server deployment.
  • Mọi hotfix từ server quay về theo đường server → Git → local sync.
  • môi trường chạy data, session files, secrets và submission payload không đi vào nguồn thư viện.

3. Chuẩn bị local synced worktree

Worktree được phép

Ví dụ path sync được phép:

%UserProfile%\HESEM Engineering\HESEM-Digital - QMS-nguồn-kiểm soát\01-Controlled-nguồn\qms.hesem.com.vn\

Git-dir được phép

Ví dụ git siêu dữ liệu path:

C:\HESEM-Git\qms.hesem.com.vn.git\
Rule cứng: không để .git trong thư mục SharePoint sync. Không sync nguồn thư viện trên máy không được đăng ký. Không cho nhiều Người cùng dùng chung một local worktree.

4. Tạo local làm việc copy đúng chuẩn

Bước 4.1 - Đồng bộ thư viện nguồn xuống máy

  1. Đăng nhập OneDrive bằng tài khoản công ty được cấp quyền.
  2. Mở QMS-Source-Control và chọn Sync.
  3. Xác nhận thư mục local xuất hiện đúng dưới profile Người dùng.

Bước 4.2 - Clone với separate git-dir

Chỉ thực hiện khi thư mục worktree đang trống:

git clone --separate-git-dir C:\HESEM-Git\qms.hesem.com.vn.git <git-remote-url> "%UserProfile%\HESEM Engineering\HESEM-Digital - QMS-nguồn-kiểm soát\01-Controlled-nguồn\qms.hesem.com.vn"

Nếu worktree đã có dữ liệu sync sẵn, IT phải tạo lại worktree sạch trước khi clone.

Bước 4.3 - Kiểm tra ranh giới

  1. Chạy git status trong worktree.
  2. Xác nhận thư mục worktree không có .git.
  3. Xác nhận git rev-parse --git-dir trỏ về C:\HESEM-Git\....

5. Chu trình chuẩn local → Git → server

BướcHành động bắt buộcBằng chứng

6. Chu trình hotfix server → Git → local

BướcHành động bắt buộcBằng chứng
1Khi sửa trực tiếp trên server, mở reverse-sync intake ngay.ReverseSyncId
2Commit và push thay đổi từ server về Git remote.CommitId
3Local admin endpoint pull về worktree đang sync.Local pull log
4Kiểm tra SharePoint sync hoàn tất.Sync complete bằng chứng
5Đóng reverse-sync sổ đăng ký.Close-out hồ sơ
Rule khẩn: không giữ thay đổi meaningful trên server mà chưa commit back quá một ca làm việc. Không dùng server như nơi soạn thảo chính.

8. trực tuyến biểu mẫu boundary

  • Form definition, released schema, nguồn template và supporting nguồn code đi theo nguồn cycle của WI-107.
  • Form submissions, phê duyệts, operational responses và hồ sơ payload đi về hồ sơ libraries hoặc môi trường chạy hồ sơ areas theo WI-101.
  • luồng công việc môi trường chạy files trong qms-data/form-luồng công việc không được coi là nguồn artifact.

9. Bằng chứng bắt buộc cho mỗi phát hành

  • CommitId và GitRef
  • phát hành manifest
  • Deploy receipt
  • Smoke test hoặc point-of-use verification
  • Rollback readiness
  • Reverse-sync close-out nếu có hotfix

10. Không được go-live khi

  • Worktree local chưa tách khỏi git-dir
  • nguồn thư viện vẫn mở external sharing
  • môi trường chạy noise còn bị stage vào commit
  • Deploy không truy được manifest và smoke bằng chứng
  • Server hotfix chưa có reverse-sync close-out
