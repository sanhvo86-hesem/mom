## Scope rule

A root is not implementation-ready because it appears in the catalog. It becomes a candidate for work-package detailing only when owner, workflow command, API family, frontend action, evidence/audit/signature rule, telemetry, rollback/restriction and stop-rule are present or explicitly waived by owner.
