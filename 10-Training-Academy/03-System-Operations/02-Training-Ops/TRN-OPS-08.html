Module TRN-OPS-08

Configuration & Revision Control for CNC (Wrong‑Rev Prevention)

Mục tiêu: giảm “wrong revision / wrong program” về 0 bằng 3 lớp kiểm soát: (1) Baseline Package (đúng file, đúng Rev), (2) Gate checks (Planning release → Job kickoff → First piece), (3) Configuration audit (đối chiếu as‑built ↔ requirement).

Thông tin triển khai
  • Đối tượng (bắt buộc): Programmer/CAM, Setup/Leader, QC/QA (CMM), Planner/Dispatcher, Supervisor.
  • Thời lượng: 60’ lý thuyết + 60’ workshop trên 1 job thật + 15’ quiz.
  • Tài liệu nền: SOP‑QMS‑020, PROC‑CNC‑002, FRM‑QA‑017, WI‑OPS‑001.
Thông điệp: Nếu không chứng minh được đúng Rev → STOP/HOLD.

3) Controls thực chiến (baseline + gates + audit)

(1) Baseline Package

  • Tập file “đúng Rev” được phát hành 1 lần, có manifest/checksum.
  • Không dùng file rời (email/zalo/USB) để chạy sản xuất.
  • Được đối chiếu tại: Planning release → Job kickoff → First piece.

(2) Configuration Audit

  • Đối chiếu requirement ↔ program/setup/inspection ↔ chứng chỉ ↔ as‑built.
  • Thực hiện nhanh bằng FRM‑QA‑017.
  • Áp dụng tối thiểu cho job có: Rev mới, CTQ, outsource special process, hoặc khách yêu cầu.

(3) Gate enforcement

Gates là cơ chế cưỡng chế không bypass. Tham chiếu WI‑OPS‑001.

  • G0: Contract freeze (đúng bản vẽ/rev, CTQ nhận diện)
  • Planning Release: FRM‑PLA‑001 đối chiếu baseline manifest
  • G1: FRM‑OPS‑008 + setup sheet + program release
  • G2: First piece/FAI + (nếu có) FRM‑QA‑016 CMM program release
  • G4/G5: Config audit + ship pack completeness

4) Checklist 10 điểm (job‑level)

  1. Part/Rev trên PO = Part/Rev trên drawing/spec?
  2. Baseline package có manifest/checksum và đúng Part/Rev?
  3. Setup sheet hiển thị Program ID/Rev/Tool list/zero point?
  4. Chương trình trên máy = program phát hành (ID/Rev)?
  5. Inspection plan có CTQ method + gage “green”?
  6. Nếu dùng CMM: FRM‑QA‑016 pass và báo cáo đo có Program ID/Rev?
  7. Nếu outsource: APL/scope/cert trace đầy đủ?
  8. Traveler/labels phản ánh đúng Rev & trace lot?
  9. Job Dossier đặt đúng folder & naming?
  10. Nếu phát hiện mismatch ở bất kỳ điểm nào → STOP/HOLD & escalate.

5) Workshop: làm config audit trong 15 phút

Chọn 1 job thật (hoặc job mẫu), thực hiện FRM‑QA‑017 theo nhóm. Mục tiêu: tìm ít nhất 2 điểm mismatch tiềm ẩn và đề xuất control.

  • Nhóm 1: requirement (drawing/spec/CTQ)
  • Nhóm 2: program/setup (CAM/NC/setup sheet)
  • Nhóm 3: inspection/records (IR/FAI/CMM program)
  • Nhóm 4: ship pack (labels/CoC/certs)

6) Đánh giá (quiz + scenario)

Quiz (10 câu — bắt buộc)

  1. “One source of truth” nghĩa là gì trong config?
  2. Nêu 3 điểm dễ sai Rev nhất trong CNC.
  3. FRM‑QA‑017 dùng khi nào?
  4. Tại sao cần hiển thị Program ID/Rev trên báo cáo đo?
  5. Khi thấy program trên máy không khớp setup sheet, bạn làm gì?

Scenario

Giảng viên đưa 1 bộ job pack có 1 lỗi “mismatch” (ví dụ: drawing Rev B, setup sheet Rev A). Học viên phải phát hiện và nêu đúng gate/record để HOLD.

Pass: phát hiện mismatch + nêu đúng gate + tạo đúng bằng chứng (QC hold/log). Fail: đề xuất “chạy tạm rồi sửa sau”.
