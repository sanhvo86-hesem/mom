# HESEM MOM ERP — Mandatory AI Workflow

> **This workflow is MANDATORY for ALL AI agents.**
> Claude, GPT, Codex, Copilot, Cursor, Windsurf, Aider — no exceptions.
> **Do NOT skip any phase. Do NOT start coding before Phase 2 is complete.**

---

## Phase 0 — ORIENT (Every session, every task, no exceptions)

Before doing ANYTHING else, read these three files in order:

```
1. Read  .ai/CONVENTIONS.md      → file placement rules (WHERE to put things)
2. Read  .ai/repo-map.json        → project topology (WHAT exists and WHERE)
3. Read  AGENTS.md                → governance rules (WHAT is allowed)
```

**Why mandatory:** Skipping orientation is the #1 cause of:
- Files created at the wrong location
- Reports scattered outside `_reports/`
- Documentation placed inside `mom/` (served app content)
- Architectural decisions made without understanding the system

---

## Phase 1 — LOCATE (Before reading any source file)

Use the index. Do NOT browse directories blindly.

```
Find a table:     Grep .ai/db-map/index.json → read .ai/db-map/<domain>.json
Find a class:     Grep .ai/symbols.json (do NOT read the full file)
Find a route:     Grep .ai/route-map.json
Find a domain:    Read .ai/module-summaries/<domain>.md
Find a contract:  Grep .ai/contracts-map.json
```

**Rule:** If you cannot find what you need in the index, ask before scanning directories.

---

## Phase 2 — PLAN (Before writing any file or code)

State explicitly before starting:

1. **Domain affected** — which of the 12 domains? (master_data, planning_production, quality_improvement, etc.)
2. **Tables involved** — which tables will be read or written?
3. **Files to create** — for EACH new file, state the exact path and verify it against `CONVENTIONS.md`
4. **Regression surface** — what else might break?
5. **Tests to run** — which test commands verify the change?

**File placement check (required for every new file):**

| You are creating... | Correct location |
|---|---|
| PHP controller | `mom/api/controllers/` |
| PHP service | `mom/api/services/` |
| SQL migration | `mom/database/migrations/NNN_name.sql` |
| Domain contract | `mom/contracts/objects/<domain>--<resource>/` |
| AI audit/report generated by AI | `_reports/<category>/` |
| Architecture decision | `docs/architecture/` |
| Technical spec | `docs/backend/` or `docs/architecture/` |
| Security audit | `docs/audits/` |
| Benchmark/gap analysis | `docs/benchmark/` |
| Design system notes | `docs/design-system/` |
| AI prompt/context doc | `docs/ai-prompts/` |
| Release manifest | `mom/release/` |
| Registry report | `mom/data/registry/` |

**FORBIDDEN locations for new files:**
- ❌ Repo root (`/`)
- ❌ `mom/docs/` root (only subdirs: forms/, training/, system/, operations/, glossary/)
- ❌ Inside `mom/docs/forms/`, `mom/docs/training/`, `mom/docs/system/`, `mom/docs/operations/` unless it is actual app-served QMS content

---

## Phase 3 — EXECUTE

### Code changes
- `declare(strict_types=1)` on every PHP file
- `final class` for all services and controllers
- Every infrastructure call (Redis, RabbitMQ, S3, Loki) needs a file-based fallback
- Follow CacheService pattern: try → catch Throwable → fallback → log

### File creation
- Create the file EXACTLY at the path stated in Phase 2
- Check: is this file in a directory that matches its type?
- Check: does this file belong to `mom/` (app code/content) or `docs/` (reference only)?

### Never do
- Never create documentation at the root directory
- Never put a generated report inside `mom/docs/`
- Never mix development artifacts with app-served content
- Never put AI execution artifacts, tranche reports, or audit logs in `mom/docs/system/` root — use `mom/docs/system/agent-reports/` or `_reports/`

---

## Phase 4 — VERIFY

Run after every change:

```bash
# PHP static analysis
composer --working-dir=mom run analyse

# Unit + integration tests
composer --working-dir=mom run test

# Regenerate AI index (if PHP files, contracts, or migrations changed)
php tools/scripts/ai-index/generate.php --verbose
```

**Final checklist before declaring done:**
- [ ] No new files at repo root
- [ ] No documentation placed inside `mom/docs/` root (only in subdirs)
- [ ] No reports scattered outside `_reports/`, `mom/data/registry/`, or `mom/release/`
- [ ] No `mom/docs/ai-prompts/`, `mom/docs/backend/`, or similar development dirs inside app content
- [ ] PHPStan: 0 errors
- [ ] PHPUnit: all tests pass (no new failures)

## Report Naming Convention

All AI-generated reports must follow this naming pattern:

```
<type>-<scope>-<date>.{json|md}

Examples:
  audit-quality-domain-2026-04-15.md        ← domain audit
  report-schema-authority-2026-04-15.json   ← schema report
  closure-tranche19-2026-04-15.md           ← closure report
  gap-analysis-mes-world-class-2026-04-15.md ← benchmark

NOT allowed:
  WORLD_CLASS_CLOSURE_20260415.md   ← SCREAMING_SNAKE_CASE
  module-builder-ultra-round16.md   ← wrong location (belongs in docs/design-system/)
  wave7-report.json                 ← ok location (mom/data/registry/), ok name
```
