ANNEX-116 — Bản thiết kế ShareFile local working copy và vòng lặp đồng bộ khép kín

Ranh giới tài liệu: ANNEX này khóa cấu trúc ShareFile local, không thay thế SOP-101 về phê duyệt/phát hành, không thay thế WI-107 về thao tác từng bước, và không thay thế ANNEX-133 về SSOT M365/SharePoint.

1. Kiến trúc kiểm soát bắt buộc

Node 1 — ShareFile controlled local

Nơi người tích hợp chính soạn thảo trong working copy có đồng bộ, gom review pack, release evidence và restore point.

Node 2 — GitHub main

Sổ cái commit trung tâm để local và cPanel cùng truy về một lịch sử phiên bản.

Node 3 — cPanel live repo

Bản point-of-use đang chạy portal; pull/push từ portal admin đều vận hành trên node này.

ShareFile controlled local | v Git working copy (single active writer) | v GitHub origin/main | v cPanel live repository | v Portal point-of-use Emergency path: cPanel edit -> Push to Git -> GitHub origin/main -> local git pull -> ShareFile evidence update

2. Root folder bắt buộc trong ShareFile

Root đề nghị dùng một tên duy nhất ở vùng Shared Folders, ví dụ HESEM-QMS-Control. Tất cả thư mục con bên dưới phải dùng English slug/ngữ cảnh ngắn, không đặt tên dài gây vượt giới hạn path.

5. Quy tắc quyền và kế thừa quyền

Quyền ở root

  • Chỉ IT Administrator và QMS Engineer được tạo root-level folder mới.
  • Quyền root phải được thiết kế từ trên xuống, vì thư mục con sẽ kế thừa quyền từ thư mục cha.
  • Không cấp Admin rộng cho end user ở vùng release evidence hoặc working repo.

Personal Folder

  • Personal Folder có thể tồn tại cho nhu cầu cá nhân, nhưng không được dùng làm SSOT cho tài liệu QMS.
  • Nếu cần file cá nhân để chuẩn bị trước khi chia sẻ, phải chuyển vào vùng Shared Folders trước khi bước vào review hoặc phát hành.

8. Nội dung bắt buộc của release evidence

  • Mã DCR và phạm vi thay đổi.
  • Tài liệu bị ảnh hưởng và quyết định version.
  • Commit hash trên GitHub origin/main.
  • Bằng chứng cPanel pull hoặc portal sync thành công.
  • Ảnh/smoke test point-of-use hoặc checklist triển khai.
  • Nếu là hotfix cPanel, phải có dấu vết local pull về ShareFile sau đó.

9. Điều cấm bắt buộc

  • Không dùng ShareFile Personal Folder làm nơi giữ bản chính thức.
  • Không cho nhiều người cùng chỉnh trực tiếp một working copy Git dưới ShareFile.
  • Không để runtime file hoặc log tạm đi vào vùng snapshot/release evidence.
  • Không dùng 99-Not-Controlled-Do-Not-Use-As-SSOT như nơi phát hành.
