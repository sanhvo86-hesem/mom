ÁP DỤNG CHO
Mọi file lưu lên SharePoint: form đã điền, evidence sản xuất, engineering baseline, hồ sơ đào tạo, audit, calibration và tài sản. Naming convention thống nhất đảm bảo traceability theo AS9100 §8.5.2 và record control theo ISO 9001 §7.5.3. Tài liệu này là SSOT cho quy tắc đặt tên file trong toàn bộ hệ thống HESEM QMS.

ANNEX-137 - Evidence and Records Naming Convention

1. Phạm vi và mục đích

ANNEX-137 quy định naming convention thống nhất cho MỌI file lưu lên SharePoint trong hệ thống HESEM QMS. Quy tắc đặt tên đảm bảo:

  • Traceability theo AS9100 §8.5.2 — mỗi file có thể truy vết ngược về job, scope, người tạo, thời điểm tạo.
  • Record control theo ISO 9001 §7.5.3 — nhận dạng, lưu trữ, bảo vệ, truy xuất, thời hạn giữ và hủy.
  • Sort & search — tên file sort theo thời gian tự nhiên, search bằng keyword trong SharePoint trả kết quả chính xác.
  • Audit readiness — đạt 60-second traceability test (xem Section 10).

Phạm vi áp dụng: Toàn bộ nhân viên HESEM, nhà thầu phụ, và bất kỳ ai upload file lên hệ thống SharePoint của HESEM. Áp dụng cho tất cả các loại file: form đã điền, evidence sản xuất, engineering baseline, hồ sơ đào tạo, audit evidence, calibration certificate, supplier records, và asset records.

8. Form version control

8.1 Form blank download — version stamp

Khi user tải form blank từ web portal:

  • Filename BẮT BUỘC có V{ver}: FRM-302_Setup-Sheet-and-Tool-List_V3.2.xlsx
  • Server inject metadata vào sheet ẩn _QMS_CONTROL:
Metadata field Mô tả Ví dụ
form_codeMã formFRM-302
versionVersion form blank3.2
checksumSHA-256 checksuma1b2c3...
downloaded_byUserID người tảiSET1
downloaded_atThời điểm tải2026-03-27T09:00:00+07:00
sourceNguồn tảiportal.hesem.com.vn

8.2 Form filled upload — version validation

  • SharePoint metadata column Form-Version phải khớp active version trong form_control_registry.
  • Nếu version cũ → FLAG Version-Valid = OBSOLETE + notify uploader.
  • Monthly audit: query files where Version-Valid = OBSOLETE → action required.

8.3 Phân biệt blank vs filled nhìn từ tên file

BLANK: FRM-{code}_{Title}_V{ver}.xlsx           ← Có Title, KHÔNG có scope/HHMM
FILLED: FRM-{code}_V{ver}_{scope}_{YYYYMMDD}_{HHMM}-{UserID}.xlsx ← Có scope + time, KHÔNG có Title

10. 60-second traceability test (AS9100 compliance)

10.1 Kịch bản 1 — FAI report cho job cụ thể

Auditor hỏi: "Show me the FAI report for job 0042, part 714XXXX rev A"

Bước Hành động Thời gian
1Mở HESEM-Job-Evidence → Job-Dossiers → 20265s
2Tìm folder JOB-2026-0042_714XXXX-REVA5s
3Filter metadata: Gate = G4-FAI hoặc search "FRM-311"5s
4Thấy: FRM-311_V1.0_JOB-2026-0042_20260327_0900-QC1.xlsx
→ Biết: FAI form V1.0, ngày 27/3/2026 lúc 9:00, QC1 điền		ngay
5Kèm: CMM_JOB-2026-0042_714XXXX-REVA-OP10_20260327_0905-QC1.csv
→ CMM data OP10, cùng ngày, cùng inspector		ngay

TỔNG: < 20 giây (yêu cầu < 60 giây)

10.2 Kịch bản 2 — Tất cả NCR cho khách hàng trong Q1
