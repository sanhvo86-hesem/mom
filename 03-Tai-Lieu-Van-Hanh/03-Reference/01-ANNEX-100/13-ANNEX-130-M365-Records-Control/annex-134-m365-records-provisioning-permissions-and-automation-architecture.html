ANNEX này điều khiển cách hệ thống tự tạo đúng chỗ lưu cho hồ sơ cấp công ty, cấp phòng ban, cấp nhân sự, cấp job và cấp hệ thống số.

ANNEX-134 - M365 hồ sơ cung cấp / cấp phát, quyền truy cập and tự động hóa kiến trúc

1. Mục đích, phạm vi và kết quả bắt buộc

ANNEX này xác lập cơ chế vận hành bắt buộc để mọi hồ sơ phát sinh trong quá trình điều hành, hỗ trợ, sản xuất, giao hàng, đào tạo, quản trị nhân sự và quản trị hệ thống đều có vị trí lưu chuẩn ngay từ lúc tạo. Kết quả bắt buộc là: không còn hồ sơ vận hành bị lưu tự phát ngoài cấu trúc chuẩn; mọi thư mục được tạo qua Requests có kiểm soát; mọi quyền / sự cho phép đi qua group mô hình; mọi hồ sơ nhóm sản phẩm có lưu giữ cơ sở, siêu dữ liệu và dấu vết kiểm toán đầy đủ.

Ranh giới tài liệu: ANNEX-133 là bản thiết kế vị trí lưu và thư mục phân loại hệ thống; ANNEX-131 là nguồn chuẩn cho field từ điển, value set, list sơ đồ cấu trúc và source-of-hồ sơ rule; ANNEX-132 là chuẩn luồng phê duyệt/ghi nhật ký; ANNEX-135 là file plan chi tiết theo phòng ban, JD và job gate; ANNEX-136 là chuẩn ranh giới SharePoint local sync, Git, server deployment, runtime và reverse sync; ANNEX-137 là quy ước đặt tên file bằng chứng và hồ sơ (HHMM-UserID pattern); ANNEX-101 là nguồn chuẩn quyền theo vai trò và nguyên tắc ít nhất quyền hạn. ANNEX-134 là tài liệu điều phối thực thi các nguồn chuẩn này thành cung cấp / cấp phát kiến trúc end-to-end.

3. Kiến trúc điều khiển cung cấp / cấp phát cấp doanh nghiệp

Yêu cầu Lớp / Tầng

Người dùng không tạo cấu trúc vận hành bằng tay ngoài bản thiết kế. Mọi Requests đi qua list điều khiển như M365-Hồ sơ-cung cấp / cấp phát-Requests, Bên ngoài-chia sẻ-Phê duyệt-Log hoặc Lưu trữ-khóa-Sổ đăng ký.

Quyết định Lớp / Tầng

Flow đọc RecordType, DeptCode, RequestType, LibraryCode, ConfidentialityClass, lưu giữ map và quyền / sự cho phép map để xác định path đầu ra, nội dung khung sườn, group truy cập và control điểm.

thực thi Lớp / Tầng

Power Automate hoặc tích hợp dịch vụ tạo thư mục/thư viện path, gán siêu dữ liệu, ngắt / nghỉ kế thừa đúng rule, tạo chia sẻ link đúng loại, log run kết quả, phát hành URL cho Người Requests và gửi cảnh báo khi run fail.

Control Lớp / Tầng: Phạm vi quản lý lưu giữ, đánh giá log, truy cập rà soát, hàng quý quyền / sự cho phép tái chứng nhận, đã hết hạn chia sẻ dọn dẹp, lưu trữ khóa rà soát và ngoại lệ kết thúc / đóng là tầng kiểm soát bắt buộc sau khi cung cấp / cấp phát hoàn tất.

7. cung cấp / cấp phát chi tiết cho toàn bộ phòng ban trong 05-Phòng ban-Hồ sơ

Mọi khu vực phòng ban được tạo theo path gốc 05-Phòng ban-Hồ sơ/{DeptCode}-{DepartmentName}/. Flow tạo trước 00-Live-Control-and-Master-Data, sau đó tạo các nhánh cố định 01-Quản trị, 02-Operations, 03-Registers-and-Logs, 04-dự án-and-Cải tiến, 05-Giao diện-and-đã phát hành-Gói, 06-Working-Transitory-and-Draft, 07-Reference-and-Received-External, 99-Lưu trữ. Bên trong các nhánh này flow tạo 00-Current, 00-Active, {YYYY}, role team workbenches, 90-Employee-WB/{EmployeeID}-{DisplayName} và subfolder set theo ANNEX-135.

Operating-data rule: live board, current queue, master list, mapping, dashboard source view, open action tracker và active project không đi vào thư mục năm. Chúng phải nằm ở 00-Live-Control-and-Master-Data hoặc nhánh 00-Current/00-Active. Bản nháp, review copy, working file, pending approval export và personal team workbench file phải nằm ở 06-Working-Transitory-and-Draft. File nhận từ khách hàng, nhà cung cấp, tiêu chuẩn, manual, convenience copy và tài liệu tham khảo phải nằm ở 07-Reference-and-Received-External. Nhánh {YYYY} chỉ dành cho outputs theo kỳ, closed-period logs, review packs, snapshot và history.
Provisioning scope: CreateDepartmentZone phải tạo đủ khung sườn department, role team workbench, named employee workbench placeholder cho từng role/JD và thư mục reference/transitory chuẩn. CreateEmployeeWorkbench phải tạo named folder dưới đúng ROLE-{RoleCode}/90-Employee-WB/{EmployeeID}-{DisplayName}. CreateJobDossier phải tạo đủ subfolder theo gate chi tiết trong ANNEX-135, không chỉ tạo gate root.
Sync-safe naming control: flow provisioning không được phát sinh các segment dài hơn bộ chuẩn trong annex này. Nếu cần thêm nhánh mới, ưu tiên route bằng metadata, list index hoặc register thay vì kéo dài tên folder. Thiết kế phải giữ biên độ cho local sync, Git worktree và file name thật của Người dùng ở các nhánh sâu.
File naming vs. permission group clarification (see ANNEX-137): File naming sử dụng pattern HHMM-UserID (ví dụ: _0830-NVA, _1400-QC1).
