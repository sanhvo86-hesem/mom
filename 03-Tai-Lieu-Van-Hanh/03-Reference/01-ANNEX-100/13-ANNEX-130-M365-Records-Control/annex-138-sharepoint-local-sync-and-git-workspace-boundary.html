ANNEX-138 - Sharepoint Local Sync and Git Workspace Boundary

1. Mục đích và phạm vi

ANNEX này khóa cấu trúc endpoint local cho controlled source của HESEM. Phạm vi gồm: worktree được SharePoint sync, git-dir tách rời, endpoint registration, hạn chế của OneDrive/SharePoint sync, cấm multi-writer trên cùng worktree, và cơ chế recovery khi thay máy hoặc hỏng sync.

Ranh giới tài liệu: WI-107 khóa thao tác hằng ngày; ANNEX-136 khóa boundary giữa source, Git, server và runtime; SOP-101 và SOP-104 khóa governance tổng thể.

2. Mapping chuẩn từ SharePoint đến local

Thành phầnPath / ví dụRule
Source library pathHESEM-Digital/QMS-Source-Control/01-Controlled-Source/qms.hesem.com.vn/Đây là nguồn SharePoint được phép sync xuống local.
Local synced worktree%UserProfile%\HESEM Engineering\HESEM-Digital - QMS-Source-Control\01-Controlled-Source\qms.hesem.com.vn\Đây là nơi user làm việc với source content.

3. Kiến trúc endpoint bắt buộc

Allowed layout

SharePoint source library -> OneDrive sync folder -> qms.hesem.com.vn (worktree only) Git metadata -> C:\HESEM-Git\qms.hesem.com.vn.git

Forbidden layout

SharePoint source library -> qms.hesem.com.vn -> .git -> node_modules runtime cache -> secrets -> session data

4. Quy tắc endpoint registration

RuleYêu cầu
Named endpointMỗi máy được sync source library phải có EndpointID, owner, OS, location và approver trong register nội bộ.
One endpoint, one primary writerMỗi worktree chỉ có một Người chịu trách nhiệm chính. Không dùng shared desktop hoặc generic account làm máy authoring chính.
No unregistered syncKhông cho Người dùng tự bấm sync source library trên máy chưa được duyệt.
Periodic reviewHàng quý rà endpoint được sync, loại bỏ máy cũ, máy nghỉ việc, máy đổi owner.

5. Giới hạn kỹ thuật của SharePoint / OneDrive phải tuân thủ

  • Không dùng mapped network drive hoặc junction/symlink để làm nơi sync.
  • Không dùng cùng một worktree cho nhiều user luân phiên trên cùng máy mà không có handover rõ ràng.
  • Không xem SharePoint sync là công cụ merge cho Git internals.
  • Không đưa thư mục có biến động runtime cao vào vùng sync source.
Nguồn chính thức: Microsoft Support - Work with synced files in File Explorer; Microsoft Support - Restrictions and limitations in OneDrive and SharePoint.

6. Cấm multi-writer trên cùng worktree

Tình huốngQuy định
Nhiều Người cùng sửa trong một local synced worktreeBị cấm. Mỗi Người dùng phải có endpoint riêng hoặc làm việc qua Git branch/merge đúng chuẩn.
Dùng USB/external drive chứa worktreeBị cấm.
Để team sửa trực tiếp trên server thay cho local synced worktreeBị cấm, trừ hotfix khẩn có reverse-sync close-out.

7. Những gì không được sync vào source worktree

Nhóm dữ liệuXử lý
Runtime workflow binaries, sessions, ratelimit, cachesGiữ ở server/runtime path; không sync vào worktree.
Secrets, keys, server-only credentialsGiữ ở server-only protected path; không đặt trong source library.
Submission payload, approvals, operational record dataRoute về records libraries hoặc runtime record area; không đưa vào source worktree.
.git internalsĐặt ở git-dir riêng ngoài vùng sync.

8. Onboarding endpoint mới

  1. Phê duyệt endpoint theo register.
  2. Sync QMS-Source-Control xuống máy.
  3. Tạo git-dir riêng ngoài vùng sync.
  4. Clone bằng --separate-git-dir hoặc restore worktree đúng chuẩn.
  5. Chạy kiểm tra boundary và smoke sync.

9. Recovery khi thay máy hoặc hỏng sync

  1. Dừng sync trên máy cũ.
  2. Xác nhận mọi commit đã push lên Git remote.
  3. Thu hồi endpoint cũ khỏi register.
  4. Provision endpoint mới theo mục 8.
  5. Xác nhận local pull và SharePoint sync hoàn tất trước khi authoring lại.

10. Không được go-live khi

  • Worktree còn chứa .git trong vùng sync
  • Không rõ endpoint owner
  • Source library đang mở external sharing
  • Chưa chứng minh được cycle local sync → Git → server → local sync
