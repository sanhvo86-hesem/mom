ÁP DỤNG KHI
Dùng khi triển khai mới, tái cấu trúc, đào tạo định kỳ, đào tạo nhân viên mới, tái chứng nhận năng lực hoặc khắc phục lỗi sử dụng folder M365 cho toàn bộ dòng chảy CNC job order từ RFQ đến ship. Tài liệu này điều khiển cách huấn luyện mọi người hiểu cấu trúc folder, route file đúng nơi, nộp evidence đúng lúc và tìm lại hồ sơ trong thời gian điều hành cho phép.

WI-103 — Đào tạo, chứng nhận năng lực và sử dụng cấu trúc folder M365 trong vận hành CNC job order

1. Mục đích và kết quả bắt buộc

WI này khóa phương pháp hướng dẫn và đào tạo để mọi người hiểu cấu trúc folder bằng ngôn ngữ công việc, không học theo lý thuyết IT. Sau đào tạo, người dùng phải biết file nào là official record, file nào là current control, file nào là draft, file nào là reference, file nào phải đi vào job dossier, file nào phải ở Part master, file nào phải ở People/System restricted, và khi nào phải promote hoặc cull.

KẾT QUẢ BẮT BUỘC: người dùng route đúng file ngay tại thời điểm phát sinh; supervisor kiểm được trong ca; QA/QMS truy lại evidence active job trong 60 giây; file không còn nằm rải rác ở desktop, Downloads, Zalo, Teams chat riêng, email cá nhân, USB hoặc folder tự đặt tên.
Rule gốc: đào tạo không dừng ở “đã học”. Người dùng chỉ được xem là đạt khi làm đúng trên công việc thật, lưu đúng folder thật và truy lại được hồ sơ thật.

Kết luận áp dụng: mô hình đào tạo của HESEM là mô hình role-based, scenario-based, work-at-the-moment, assessment-based. Đây là suy luận triển khai trực tiếp từ NARA + ISO + Microsoft, không phải đào tạo nhận biết chung chung.

3. Mô hình nhận thức đơn giản mà mọi người phải nhớ

1. File cho đúng 1 Job thì vào 03-Job-Dossiers
2. File là baseline Part/Rev thì vào Part-REV-Master
3. File là current control/log/queue của phòng ban thì vào 05-Department-Records
4. File là draft/review/export tạm thì vào 06-Working-Transitory-and-Draft
5. File nhận từ khách / NCC / tiêu chuẩn thì vào 07-Reference-and-Received-External
6. File là restricted people record thì vào 08-People-Records
7. File là admin/config/access/backup thì vào 09-Digital-System-Records
8. File đã thành evidence chính thức thì phải promote khỏi draft zone

Ba câu hỏi route file trong 10 giây

  • File này phục vụ job cụ thể, phòng ban hay master part?
  • File này là official/current, draft hay reference nhận từ ngoài?
  • File này phát sinh ở gate nào, role nào, thời điểm nào?

Năm lỗi nhận thức phải dập tắt

  • “Cứ up lên SharePoint là đúng.”
  • “File Excel thì để chung một chỗ.”
  • “Ảnh để trong điện thoại, cuối tuần mới nộp.”
  • “Bản này mới nhất nên cứ ghi đè lên bản cũ.”
  • “Tìm không ra thì hỏi người khác gửi lại.”

8. Phương pháp huấn luyện phải dùng ngoài hiện trường

8.1 Classroom ngắn - không quá tải

  • Chỉ dạy mental model, rule route file, cấu trúc chính và các lỗi cấm.
  • Mỗi nhóm vai trò học đúng phần của họ, không nhồi toàn bộ hệ thống.
  • Dùng ví dụ thật từ job đang chạy hoặc job vừa ship.

8.2 Demo ngay trên file thật

  • Trainer mở đúng site, đúng folder, đúng QR/link, đúng form và làm mẫu.
  • Phải chỉ rõ vì sao file vào đây và vì sao không vào nơi khác.
  • Phải làm cả tình huống đúng và tình huống sai phổ biến.

8.3 OJT trên job thật

  • Người học tự nộp file trong ca, trainer/champion quan sát.
  • Supervisor phải kiểm retrieval ngay sau khi nộp.
  • Không tính pass nếu chỉ làm trên file demo.

8.4 Quick support tại điểm dùng

  • Dán QR hoặc link ngắn ở máy, phòng QA, receiving, shipping, control tower.
  • Mỗi QR dẫn tới đúng WI, đúng form, đúng folder example, không dẫn tới portal chung.
  • Có office hours cố định trong 2 tuần đầu sau go-live.
Không dùng phương pháp này: gửi một email chứa link ANNEX và yêu cầu mọi người tự đọc. Cấu trúc folder M365 là kỹ năng thao tác và quyết định nghiệp vụ, không phải tài liệu đọc hiểu đơn thuần.

11. Cơ chế duy trì sau đào tạo

Nhịp kiểm hàng ngày

  • Mỗi supervisor kiểm ít nhất 3 hồ sơ mới mỗi ca.
  • Tier meeting ghi lỗi route file vào FRM-208.
  • Lỗi critical phải sửa trong cùng ngày.

Nhịp review hàng tuần

  • QMS + department champion sample ít nhất 10 file/tuần/department.
  • Review workbench 06 và reference zone 07.
  • File tồn quá hạn phải promote, archive hoặc cull.

12. Triggers phải đào tạo lại

  • Nhân viên mới, đổi ca, đổi role, đổi supervisor hoặc đổi line responsibility.
  • Đổi cấu trúc folder, metadata, naming rule, permissions hoặc QR/link routing.
  • Lỗi critical: lưu sai hồ sơ khách hàng, mất evidence, lưu restricted record sai chỗ, dùng draft làm bản chính.
  • Kết quả audit retrieval hoặc live routing dưới chuẩn trong 2 kỳ liên tiếp.
  • Thay đổi WI/SOP/ANNEX ảnh hưởng gate, form, evidence set hoặc Part master logic.
