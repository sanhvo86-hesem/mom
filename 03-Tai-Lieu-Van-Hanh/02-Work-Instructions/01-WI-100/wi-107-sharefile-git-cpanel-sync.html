WI-107 - M365 SharePoint local sync, Git và server deployment control

🗺️ Vị trí trong hệ thống 8 cổng (G0→G7): WI này vận hành xuyên suốt (G0→G7). Đồng bộ ShareFile-Git-cPanel.
→ Xem ANNEX-502 để tra bảng cross-reference.

1. Kết quả phải đạt

  • Controlled source nằm trong 10-QMS-Source-Control/01-Controlled-Source/qms.hesem.com.vn/ trên SharePoint.
  • Local working copy được sync bởi OneDrive/SharePoint, nhưng thư mục .git nằm ngoài vùng sync.
  • Mọi thay đổi source đi theo đường SharePoint local sync → Git → server deployment.
  • Mọi hotfix từ server quay về theo đường server → Git → local sync.
  • Runtime data, session files, secrets và submission payload không đi vào source library.

2. Điều kiện tiên quyết

Điều kiệnYêu cầu

3. Chuẩn bị local synced worktree

Worktree được phép

Ví dụ path sync được phép:

%UserProfile%\HESEM Engineering\HESEM-Digital-Control - 10-QMS-Source-Control\01-Controlled-Source\qms.hesem.com.vn\

Git-dir được phép

Ví dụ git metadata path:

C:\HESEM-Git\qms.hesem.com.vn.git\
Rule cứng: không để .git trong thư mục SharePoint sync. Không sync source library trên máy không được đăng ký. Không cho nhiều người cùng dùng chung một local worktree.

4. Tạo local working copy đúng chuẩn

Bước 4.1 - Đồng bộ library source xuống máy

  1. Đăng nhập OneDrive bằng tài khoản công ty được cấp quyền.
  2. Mở 10-QMS-Source-Control và chọn Sync.
  3. Xác nhận thư mục local xuất hiện đúng dưới profile người dùng.

Bước 4.2 - Clone với separate git-dir

Chỉ thực hiện khi thư mục worktree đang trống:

git clone --separate-git-dir C:\HESEM-Git\qms.hesem.com.vn.git <git-remote-url> "%UserProfile%\HESEM Engineering\HESEM-Digital-Control - 10-QMS-Source-Control\01-Controlled-Source\qms.hesem.com.vn"

Nếu worktree đã có dữ liệu sync sẵn, IT phải tạo lại worktree sạch trước khi clone.

Bước 4.3 - Kiểm tra ranh giới

  1. Chạy git status trong worktree.
  2. Xác nhận thư mục worktree không có .git.
  3. Xác nhận git rev-parse --git-dir trỏ về C:\HESEM-Git\....

5. Chu trình chuẩn local → Git → server

BướcHành động bắt buộcEvidence
1Soạn/chỉnh source trong worktree local đang sync.Changed files
2Kiểm tra diff, loại runtime noise khỏi commit.Clean git status
3git add → git commit → git push.CommitId
4Đăng ký release manifest theo ANNEX-132.Manifest URL
5Server operator pull đúng commit/tag, chạy smoke test, lưu deploy receipt.Deploy receipt

6. Chu trình hotfix server → Git → local

BướcHành động bắt buộcEvidence
1Khi sửa trực tiếp trên server, mở reverse-sync intake ngay.ReverseSyncId
2Commit và push thay đổi từ server về Git remote.CommitId
3Local admin endpoint pull về worktree đang sync.Local pull log
4Kiểm tra SharePoint sync hoàn tất.Sync complete evidence
5Đóng reverse-sync register.Close-out record
Rule khẩn: không giữ thay đổi meaningful trên server mà chưa commit back quá một ca làm việc. Không dùng server như nơi soạn thảo chính.

8. Online forms boundary

  • Form definition, released schema, source template và supporting source code đi theo source cycle của WI-107.
  • Form submissions, approvals, operational responses và record payload đi về records libraries hoặc runtime record areas theo WI-101.
  • Workflow runtime files trong qms-data/form-workflow không được coi là source artifact.

9. Evidence bắt buộc cho mỗi release

  • CommitId và GitRef
  • Release manifest
  • Deploy receipt
  • Smoke test hoặc point-of-use verification
  • Rollback readiness
  • Reverse-sync close-out nếu có hotfix

10. Không được go-live khi

  • Worktree local chưa tách khỏi git-dir
  • Source library vẫn mở external sharing
  • Runtime noise còn bị stage vào commit
  • Deploy không truy được manifest và smoke evidence
  • Server hotfix chưa có reverse-sync close-out
