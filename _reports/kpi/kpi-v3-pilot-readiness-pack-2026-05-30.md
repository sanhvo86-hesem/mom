## Part A — Vietnamese expert rewrite (plan + glossary)

The rewrite targets are portal-managed HTML (ANNEX-110/122/127/128-source/129,
WI-202, JD §9) whose content is authored through the portal document editor; the
pre-commit doc-guard blocks local commits of these files (HESEM policy — the VPS
runtime owns them, and a local overwrite risks losing portal-authored edits).
Therefore the rewrite is delivered as **(a)** the registry `name_vi` / VN prose
fields that the renderers read (already expert-VN in the entries written this
program), and **(b)** this plan + locked glossary for the portal-side rewrite.

**Invariant:** do not change any KPI code, target, threshold, owner, or formula —
language only.

### Pilot rules (30 days)

- Each KPI appears in ≥1 T1/T2/T3 forum.
- Each red KPI produces an action record (owner + due + evidence).
- Each manual input has evidence + approval (pending→verified).
- Each red counter is reviewed beside its primary.
- Each gate hold has reason + evidence + release rule.
- Each small-sample case → event review, not rate punishment.
- Each customer-critical issue → escalation.

action_owner | action_due | counter_status | evidence_ref | closed_at | learning
