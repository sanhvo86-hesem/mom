## 1. Purpose

Adversarially audit the alias resolution path for ambiguity, hijack risk, scope confusion, and quarantine-bypass. Verify that every dangerous outcome is either blocked at the DB / service layer, or surfaces in the quarantine triage queue rather than silently coercing.

## 2. Threat model

| Threat | Path | Mitigation |
|---|---|---|
| Alias hijack — supplier override of SYSTEM alias for same text | resolver favours SUPPLIER over SYSTEM | acceptable when supplier_id is verified upstream; supplier_id auth ties to user/session |
| Alias collision — two suppliers claim the same alias text for different canonicals | composite unique key `(alias_code, context_scope, COALESCE(supplier_id,''))` | both rows coexist; resolver picks the matching supplier_id |
| Quarantine bypass — service silently coerces unresolvable input | every alias miss raises `UOM_EXTERNAL_CODE_UNKNOWN` + queues quarantine row | enforced by `UomAliasResolutionService::resolve` contract |
| Quarantine exhaustion — attacker spams unresolvable aliases to bloat queue | rate-limit middleware + admin pagination | additional: AI advisor can pre-triage with `ai_suggested=true` |
| Stale alias — alias points to retired canonical | resolver checks lifecycle on the canonical; raises `UOM_UNIT_NOT_ACTIVE` | confirmed in compatibility rules |
| Scope confusion — caller submits SUPPLIER scope but no supplier_id | DB CHECK rejects | confirmed |
| LIMS μm ambiguity (Length vs SurfaceRoughness) | resolver dispatches via quantity_kind hint | documented |
| Case sensitivity attack — `MM` vs `mm` | aliases stored case-sensitively but lookup uses `lower()` index for case-insensitive lookup; canonical resolution always lowercase | acceptable; documented in CD-001 |
| Effective-date back-dating attack | timestamps generated by server, not client | acceptable |

## 3. Findings

| Severity | ID | Finding | Repair |
|---|---|---|---|
| medium | AT-001 | SUPPLIER scope alias has higher priority than SYSTEM — risk if supplier auth is compromised | document in lattice; future: supplier auth strength check |
| medium | AT-002 | Quarantine queue can grow unbounded if AI advisor errors on every resolve | rate-limit per (alias_text, scope, supplier_id) — suppress duplicates within 24h |
| medium | AT-003 | LIMS μm dispatch by quantity_kind hint requires caller to know the hint; if absent, resolver defaults to length, not roughness | document; surface in API contract |
| low | AT-004 | Alias triage actor (`reviewed_by`) currently FK to users; no separate metrology-role enforcement at DB layer | enforce via app-level role check |
| low | AT-005 | Customer-scoped alias path defined but not yet exercised in unit tests | extend |
