## Harsh 10-Angle Critique

1. Controllability is now explicit, but it is still mostly registry-governed text. The next hard step is tying red actions to real corrective-action, OJT or daily-management events.
2. The renderer is better, but it remains a hydration layer. If an authenticated API session is absent, the old static section remains instead of showing the controlled registry version.
3. Admin Console now exposes blockers/control scope, but it is still a summary surface, not a full role-scorecard editor with guided validation.
4. Candidate and optional banks are structured, but there is no rotation scheduler or review cadence engine yet.
5. Reward suppression is enforced for role measures, but managers still need a separate calibrated review policy so "not automatic reward" is not misread as "not used in review".
6. FIN keeps COPQ as a capture/completeness responsibility. That is defensible after Prompt 05, but the UI must keep explaining that FIN does not own the manufacturing defect rate.
7. JD pages render from the registry only for authenticated portal users. That matches controller comments, but direct anonymous file browsing cannot prove hydration.
8. Role blockers exist as registry lists, not executable workflow blockers with owner, due date, escalation or closure evidence.
9. The integrity guard catches Prompt 09 drift, but it does not yet evaluate the natural-language quality of each role's action-when-red text.
10. The active count problem is fixed, but role usefulness still needs real supervisor feedback after the first review cycle.
