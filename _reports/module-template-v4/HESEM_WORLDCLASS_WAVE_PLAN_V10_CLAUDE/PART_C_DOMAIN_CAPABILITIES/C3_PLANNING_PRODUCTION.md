## 1. Purpose

The Planning & Production domain bridges demand (D-01 Commercial) to execution (D-06 Shopfloor).
It answers: what should be produced, when, on which equipment, in what sequence, with what
material, and with what capacity buffer. It provides the authoritative production plan that all
downstream domains execute against.

ISA-95 alignment: this domain operates at Level 4 (enterprise planning layer — MPS, MRP,
capacity planning) and Level 3 boundary (finite scheduling, dispatch list — handoff to C6 MES).

## 4. Capabilities

### CAP-C3-01 — MPS Authoring and Feasibility Simulation

**Purpose.** Produce the time-phased master production schedule from demand inputs (confirmed
SOs, demand plan, safety stock targets). Run feasibility simulations before approval.

**Lifecycle.** Demand inputs aggregated per item per bucket. Gross-to-net calculation applied
(demand − safety stock − on-hand − open supply). MPS lines generated with source flags.
Feasibility simulation: RCCP (Rough-Cut Capacity Planning) runs against MPS draft to surface
overload at key work centers. Planning Lead reviews overloads, adjusts MPS lines (defer,
split, expedite), and approves. Approved MPS activates → triggers MRP run.

**Lot-sizing rules.** Per item: LOT_FOR_LOT (exact demand), FIXED_PERIOD (N periods of demand
consolidated), MINIMUM_ORDER_QTY (round up to supplier MOQ), EOQ (economic order quantity).
Rule configured per Item Master (C2).

**Wave target.** L4 W5; L5 W5.

**Acceptance evidence.** MPS correctly reflects SOs + forecast + safety stock. Feasibility
simulation surfaces work center overloads before approval. Lot-sizing rules applied correctly:
MOQ test — demand of 50 with MOQ=100 → MPS line = 100.

### CAP-C3-02 — MRP Engine (FEFO and Per-Pack Rules)

**Purpose.** Compute material requirements from MPS, BOM, lead times, and current inventory.
Generate planned orders (purchase or make). Apply FEFO allocation logic for shelf-life-controlled
items. Apply per-pack rules (campaign for Pharma, allergen scheduling for Food, ITAR material
segregation for Aero).

**MRP logic.** For each item in MPS:
1. Explode BOM (multi-level, per C2 BOM at effective revision).
2. Compute gross requirements per period.
3. Net against: on-hand inventory (C5) + open PO receipts (C4) + open WO completions (C6).
4. Apply lot-sizing rule.
5. Offset by lead time (procurement: item.lead_time_days; manufacturing: routing total
   standard time from C2).
6. Generate MRP Action records (PLAN_ORDER_BUY or PLAN_ORDER_MAKE).

**FEFO rule.** For items with shelf_life_days set: MRP pegs demand to oldest available lot
first (C5 FEFO query). If oldest lot will expire before demand date, MRP generates an
EXPEDITE action or a new planned order as appropriate.

**Idempotency.** MRP run is idempotent: same MPS + same inventory snapshot → same action set.
Duplicate run within same snapshot window returns cached result.

**Wave target.** L4 W5; L5 W5.

**Acceptance evidence.** Multi-level BOM explosion tested at depth ≥ 6. Lead time offset
correct per routing standard time. FEFO pegging: demand for item with 30d shelf life pegged
to lot with 25d remaining rather than 60d lot. Campaign rule: Pharma items grouped by
active ingredient + cleaning campaign in schedule output.

### CAP-C3-03 — Capacity Planning (RCCP + Detailed CRP)

**Purpose.** Assess available capacity against production plan at two levels: Rough-Cut
Capacity Planning (RCCP) for feasibility screening of MPS, and Capacity Requirements Planning
(CRP) for detailed load per work center per period.

**RCCP.** Uses bill-of-resources (key work centers × standard hours per item from routing).
Runs fast against MPS draft. Surfaces overloads at aggregate level. Used in MPS approval gate.

**CRP.** Detailed calculation per job per operation per work center. Accounts for setup time,
run time (from routing), machine efficiency (from C6 OEE history), and scheduled maintenance
windows (from C9). Updates Capacity Plan records per period.

**Calendar and shift configuration.** Capacity Plan consumes from a production calendar
(working days, holidays) and shift pattern (shifts per day, hours per shift). Both configured
per work center.

**Wave target.** L4 W5 (RCCP); L5 W5 (CRP).

**Acceptance evidence.** RCCP overload alert generated when MPS requires 120% of available
work center hours. CRP recalculates within 5 minutes of new job release. Maintenance window
reduces available_hours correctly for the affected period.

### CAP-C3-04 — Finite-Capacity Scheduling (Sequence-Aware, Campaign Mode)

**Purpose.** Generate a time-sequenced, finite-capacity schedule respecting sequence
dependencies, setup matrices, campaign constraints, and priority rules. Produce dispatch
list per work center.

**Algorithm.** Constraint-based scheduling with Theory of Constraints (TOC) buffer management.
Bottleneck work center identified from CRP (highest utilization). Schedule generated
bottleneck-first (drum). Non-bottleneck work centers buffered (rope = release timing;
buffer = inventory protection). Sequence optimization via setup matrix (minimize sequence-
dependent setup times; greedy nearest-neighbor per work center).

**Campaign mode.** For Pharma (J1) and Food (J5) tenants: items within the same campaign
group (same active ingredient or allergen profile) are scheduled in contiguous blocks to
minimize cleaning changeovers. Campaign_id field on Schedule Slot links slots of the same
campaign.

**Frozen fence.** Slots within the frozen fence (configurable: default T+24h) cannot be
rescheduled by the engine. Only Production Lead can unfreeze with documented reason.

**Wave target.** L4 W6; L5 W6.

**Acceptance evidence.** Setup matrix applied: items A→B require 2h setup; B→A require 4h;
schedule sequences A then B minimizing total setup. Campaign mode: allergen items grouped;
cleaning slot inserted between campaign blocks. Frozen fence: engine cannot move a slot
in T+12h window.

### CAP-C3-05 — APQP Phase Management (Auto — SM-APQP, J2)

**Purpose.** Manage the 5-phase APQP project lifecycle per AIAG APQP 2nd Edition. Track
phase outputs, gate reviews, and launch readiness for all new automotive product launches.

**Lifecycle.** Per SM-APQP full transition table (§3.2). Key gates: Phase 1 approval (customer
inputs accepted, targets set), Phase 3 approval (process design complete, PFD/PFMEA/Control
Plan released), Phase 4 approval (Production Trial Run approved, PPAP submitted), Phase 5
launch confirmation.

**Phase output tracking.** Checklist per phase. Each checklist item has status (not_started|
in_progress|complete|waived). Gate review blocked until all non-waived items = complete.
Waivers require Planning Lead + Quality Lead co-sign with documented justification.

**Wave target.** L4 W6; L5 W8. (J2 Auto pack only.)

**Acceptance evidence.** Gate blocked when checklist item incomplete. Phase 4 gate blocked
when SM-PRR ≠ approved. PPAP submission enabled only after Phase 4 gate passes.

### CAP-C3-06 — AS9145 APQP — Aerospace (J3)

**Purpose.** Manage 6-phase aerospace APQP per AS9145 (APQP4AERO) for new aerospace product
launches. Aligned to AS9100D §8.3 and customer requirements (Boeing, Airbus, Safran, etc.).

**Lifecycle.** Per SM-APQP with 6-phase variant. Phase 6 (Feedback and Corrective Action) is
continuous and runs in parallel with production; it closes only after sustained capability
(Cpk ≥ 1.67 for ≥ 6 months of production). First Article Inspection (FAI, SM-FAI) is a gate
within Phase 5 (Validation): FAI must reach `approved` before Phase 5 gate passes.

**Wave target.** L4 W7; L5 W8. (J3 Aero pack only.)

**Acceptance evidence.** Phase 5 gate blocked when fai_id = null or FAI status ≠ approved.
Phase 6 auto-close after 6 months of Cpk ≥ 1.67 data from C6 SPC records.

### CAP-C3-07 — Demand Sensing and Forecast Consumption

**Purpose.** Ingest short-horizon demand signals (EDI 862 customer schedules, POS data,
IoT demand pull) to update the planning picture faster than the weekly MRP cycle. Track
forecast consumption to prevent double-counting.

**Lifecycle.** Demand signals arrive continuously. Demand Sensing module compares signals
against current Demand records and updates. Forecast Consumption Records updated as
SOs are confirmed (reducing forecast_qty_remaining). Planning Lead reviews net demand
picture daily; approves emergency MRP re-run when demand delta exceeds configurable
threshold (default: > 10% change in a period).

**Wave target.** L4 W5 (forecast consumption); L5 W6 (demand sensing integration).

**Acceptance evidence.** Forecast consumption: SO confirmed for 100 units → forecast_qty_remaining
decreases by 100. Demand sensing: EDI 862 ship schedule arrives; demand record updated within
15 minutes of EDI parse.

### CAP-C3-09 — Annual Layout Inspection Scheduling (Auto — J2)

**Purpose.** Schedule and track Annual Layout Inspections (ALI) per IATF 16949 §8.6.2.
Ensure every active automotive production part is dimensionally verified against its drawing
annually or on engineering change.

**Lifecycle.** ALI records auto-created: (a) 12 months after prior ALI completion; (b) when

triggers ECO in C2 if drawing discrepancy.

**Wave target.** L4 W6; L5 W8. (J2 Auto pack only.)

**Acceptance evidence.** ALI auto-created 12 months after prior completion (tested by advancing
system date). ALI auto-created when ECO closes for the item. Failed ALI automatically opens
SCAR candidate in C4.

### CAP-C3-10 — Per-Pack Scheduling Overlay

**Purpose.** Apply pack-specific scheduling constraints that override the base finite schedule
logic: campaign scheduling for Pharma/Food, allergen segregation for Food, ITAR material
segregation for Aero, sterilization cycle booking for MD.

**Pharma campaign (J1).** Items sharing an active ingredient or an open cleaning validation
campaign are grouped. Schedule engine ensures items within a campaign run contiguously with
a cleaning cycle slot inserted between campaigns. Campaign scheduling uses SM-CLEANING-V
cycle status (M4 pack SM) to determine which cleaning protocol is current.

**Allergen segregation (J5 Food).** Items with conflicting allergen profiles cannot share
a work center in the same shift without a validated allergen cleaning event between them.
Schedule engine enforces separation or inserts a validated allergen cleaning slot. Allergen
conflict matrix configured per site; defaults from item.allergen_flags comparison.

**ITAR material segregation (J3 Aero).** Items manufactured from ITAR-controlled materials
or containing ITAR-controlled components must be scheduled in ITAR-designated work areas.
Schedule engine assigns slots only to work centers flagged itar_authorized = true (C6 Work
Center entity field). Mixing ITAR and non-ITAR jobs in same slot is a scheduling violation.

**Sterilization cycle booking (J4 MD).** Items requiring terminal sterilization are scheduled
with a sterilization cycle slot reserved in the clean room / sterilization chamber work
center. Sterilization cycle has a fixed duration (ETO: 12-24h; gamma: 2-4h; steam: per cycle).
Schedule engine buffers upstream production to complete before the sterilization window.

**Wave target.** L4 W6 (campaign); L5 W8 (all overlays).

**Acceptance evidence.** Campaign violation test: scheduling engine attempts to schedule two
different-campaign Pharma items back-to-back without cleaning slot → blocked; cleaning slot
auto-inserted. ITAR violation test: ITAR job assigned to non-ITAR work center → schedule
validation error. Allergen segregation test: items with tree nut + peanut allergens not scheduled
consecutively without allergen clean event.

## 11. Boundary with Adjacent Domains

- **D-01 Commercial**: Confirmed SOs are the firm demand tier in MPS. Demand Plan (C1)
  is the unconstrained forecast input. ATP (available-to-promise) dates from Capacity Plan
  feed back to C1 quote lead-time validation.

- **D-02 Engineering**: BOM revisions and Routing standard times are primary MRP inputs.
  ECO changes to BOM/Routing trigger MRP re-run. APQP Phase outputs (PFMEA, Control Plan)
  are authored in C2/C7 and referenced from C3 APQP Phase checklist.

- **D-04 Procurement**: MRP PLAN_ORDER_BUY actions generate purchase requisitions in C4.
  Open PO receipts from C4 are a supply input to MRP netting. APQP projects may involve
  supplier APQP sub-projects in C4.

- **D-05 Inventory**: On-hand inventory from C5 is the primary inventory input to MRP
  netting. Lot availability and FEFO state from C5 drive FEFO-pegged MRP actions.

- **D-06 Shopfloor/MES**: Finite schedule drives job release in C6. Dispatch list consumed
  by C6 Dispatch Board (HMV4 Slice 1). Actual production times from C6 feed back into
  Capacity Plan variance and Cpk calculation for PTR.

- **D-09 Maintenance**: Equipment availability windows (planned maintenance) from C9 reduce
  available_hours in Capacity Plan for affected periods and work centers.

## 12. Per-Pack Overlays

### J1 — Pharma

- Campaign scheduling enforced: items sharing active ingredient or cleaning validation
  campaign scheduled contiguously. Cleaning cycle slot duration from SM-CLEANING-V.
- ICH Q10 §3.2.1: manufacturing process monitoring drives monthly planning review meeting.
  Any Cpk < 1.33 trend triggers process capability investigation before next MPS cycle.
- DSCSA: supply plan for drug products includes serialization batch allocation for
  EPCIS commissioning (link to C8 Traceability).

### J2 — Automotive

- APQP Phase Management (CAP-C3-05) mandatory for all new part launches.
- Production Trial Run (CAP-C3-08) mandatory before PPAP submission.
- ALI (CAP-C3-09) mandatory annually and on drawing change.
- Customer-specified launch timing (PPAP timing per customer CSR) tracked against APQP
  Phase 4 gate date. Late PPAP alert to Commercial Lead 30 days in advance.

### J3 — Aerospace

- AS9145 APQP (CAP-C3-06) mandatory for all new part launches with aerospace customers.
- ITAR material segregation enforced in schedule (CAP-C3-10): ITAR-designated work areas only.
- FAI (SM-FAI in C7) is a Phase 5 gate. Schedule engine reserves FAI inspection slot
  in the schedule after first production completion.

### J4 — Medical Device

- Sterilization cycle booking (CAP-C3-10): sterilization work center slots reserved per
  production batch. Sterilization hold time included in lead time calculation.
- Device history record (DHR) readiness: schedule engine checks that DMR (C2) is released
  before releasing a job for a regulated device item.

### J5 — Food & Beverage

- Allergen segregation scheduling enforced (CAP-C3-10). Allergen cleaning event duration
  per validated cleaning procedure (C7 Controlled Document).
- FSMA HARPC: production schedule considers CCP monitoring intervals. Schedule engine does
  not schedule production across a CCP monitoring window without operator confirmation.

## 13. Failure Modes Catalog

| Failure | Trigger | Impact | Recovery |
|---|---|---|---|
| MRP double-run | Two MPS approval events fire simultaneously | Duplicate planned orders | MRP idempotency: run locked per MPS version; second trigger returns cached result |
| Frozen slot violated by rescheduling engine | Bug in schedule engine | Production disruption | Frozen flag checked before any slot move; engine returns validation error if frozen |
| Campaign violation in schedule | Schedule engine does not apply allergen matrix | Cross-contamination risk | Per-pack overlay validation runs as post-generation assertion; schedule with violations rejected |
| APQP gate bypass | Checklist item waived without proper approval | Regulatory audit finding | Waiver requires dual sign: Planning Lead + Quality Lead; waiver stored on phase decision record |
| PTR auto-approved via API | Missing BD-18 enforcement | Invalid PPAP submission | BD-18 enforcement: POST /ptrs/{id}/approve returns 403 unless dual e-sig tokens present |
| MPS not updated after large SO change | Planning cycle too long | OTIF failure | Demand delta threshold: > 10% change in any period triggers alert to Planning Lead with 4h response SLA |

## 14. KPIs

| KPI | Formula | Target | Measurement |
|---|---|---|---|
| Schedule Adherence | (operations completed on scheduled day) / total scheduled operations | ≥ 90% | Daily; per work center; C6 actual completion vs schedule slot |
| MRP Action Acceptance Rate | accepted MRP actions / total open MRP actions | ≥ 85% | Monthly; high override rate signals MRP data quality issue |
| Capacity Utilization | planned_demand_hours / available_hours per work center | 75-90% target band | Weekly; < 70% = underload alert; > 100% = overload alert |
| APQP On-Time Gate Rate | APQP phases that pass gate on or before target date / total gates | ≥ 80% | Per APQP project; late gates escalated to Planning Lead |
| OTIF (Planning-attributed) | SOs missed OTIF where root cause = planning (insufficient capacity or MRP failure) / total OTIF misses | < 10% of OTIF misses attributable to planning | Per SO OTIF root cause analysis |
| PTR First-Pass Approval Rate | PTRs approved at first attempt / total PTRs | ≥ 75% | Per APQP project; second-attempt PTRs investigated for root cause |
