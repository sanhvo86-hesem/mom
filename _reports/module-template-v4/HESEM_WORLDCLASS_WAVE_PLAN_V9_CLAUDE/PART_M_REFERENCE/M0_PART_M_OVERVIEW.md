# PART_M — REFERENCE — Overview

Part M is the reference appendix. Other Parts cite Part M chapters
when they need a definition, a directory, or a registry without
re-stating it. This is the implementation of the no-duplicate rule
in READING_DISCIPLINE.md.

```
M1   Glossary                         every term defined once
M2   Domain Models                    all 14 domains with bounded-context map
M3   Root Catalog                     all 95 authoritative roots + ownership
M4   State Machine Directory          all 14 state machines + states + events
M5   SLO Directory                    all 22 SLOs + measurement
M6   Risk Register                    cross-program risks + mitigation
M7   Decision Phrases                 every BASELINE_LOCKED phrase indexed
M8   Standards Directory              all 18 governing standards
M9   Bibliography + Cross-Reference   citations and inter-Part link map
```

9 chapters plus this overview.

---

## How to use Part M

- **Authoring** new Parts: do not re-state a definition; cite Part M.
- **Reading** an unfamiliar term: search Part M first.
- **Validating** consistency: Part M is the single source for term,
  root, state, SLO, standard, decision phrase definitions.
