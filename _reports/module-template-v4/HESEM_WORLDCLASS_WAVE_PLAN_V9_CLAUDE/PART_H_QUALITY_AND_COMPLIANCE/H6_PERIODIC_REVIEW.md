# H6 — Periodic Review

```
chapter_purpose: bi-annual minimum review of computerised systems per
                 Annex 11 §11; ICH Q10 management review (annual)
owner_role:      Quality Lead with Compliance Lead
```

---

## 1. Cadence

```
Bi-annual (minimum):  per Annex 11 §11 review
Annual:                ICH Q10 management review (when applicable)
Quarterly:             per CS-B Continuous Validation cadence
Monthly:               metrics review + drift / freshness check
```

---

## 2. Review inputs

```
- Risk register status (PART_M5)
- Validation evidence freshness
- Access control list snapshot
- Incident log
- Change control history (ECO records)
- Customer complaints
- KPI dashboard (PART_A5)
- CAPA effectiveness data
- Supplier performance
- Training compliance
- Regulatory updates affecting QMS
```

---

## 3. Review outputs

- Action items (CAPA, resource changes, scope changes, training updates)
- Risk register updates
- Management decisions ratified
- Review record stored as evidence_artifact (permanent retention)

---

## 4. Per-tenant cadence

Some customers require annual or more-frequent reviews. HESEM supports
per-tenant cadence configuration.
