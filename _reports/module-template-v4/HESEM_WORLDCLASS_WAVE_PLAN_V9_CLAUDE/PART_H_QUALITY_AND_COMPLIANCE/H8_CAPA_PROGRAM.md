# H8 — CAPA Program

```
chapter_purpose: corrective + preventive action program; effectiveness
                 verification discipline
owner_role:      Quality Lead
```

---

## 1. Reference

The CAPA program lifecycle is fully described in D6 NC to CAPA. H8
focuses on the program-level discipline.

---

## 2. CAPA triggers

```
- Critical / major NC always
- Trend (3+ minor NCs same defect mode in 30 days)
- Customer complaint with quality root
- Internal audit finding (D13)
- External audit finding
- Regulator inspection finding
- Supplier-source NC (with SCAR)
- Field return / recall investigation
- Calibration OOT propagation
- SPC out-of-control signaling systemic issue
```

---

## 3. Effectiveness check methodology

(Per V5 file 08 §3 carry-forward)

```
- Effectiveness check window: typically 60-90 days
- Metric: zero recurrence of defect mode (or 80% reduction)
- Owner sign-off (e-sign)
- Independent reviewer co-signs (two-person for regulated)
```

---

## 4. CAPA effectiveness analytics

Tracked across the customer:
- CAPA cycle time
- CAPA effectiveness pass rate
- Repeat-CAPA rate (failed effectiveness)
- Defect mode Pareto

Surface on Quality Trend Dashboard (DL-02).

---

## 5. CAPA close authority

Two-person e-signature for regulated industries. Banned for AI per
RULE-2 (BD-3). Always human authority.
