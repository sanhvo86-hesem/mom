# H3 — Audit Program

```
chapter_purpose: how HESEM handles internal audits, customer audits,
                 regulator inspections, certification surveillance
owner_role:      Quality Lead with Compliance Lead
```

---

## 1. Audit types

```
Internal audit               per ISO 9001 §9.2; annual minimum per
                             system; per-process per IATF VDA 6.3
Customer audit               OEM, prime contractor, or end customer
                             auditing HESEM tenant
Notified body audit          EU MDR Notified Body, ISO 13485 NB
Regulator inspection         FDA, EMA, NHTSA, FAA, USDA
Certification surveillance   IATF 16949 (3-year cycle), ISO 27001,
                             ISO 13485, AS9100D
NADCAP audit                 special process accreditation (aerospace)
SOC 2 audit                  Type II observation period (annual)
Self-assessment              voluntary, periodic readiness check
```

---

## 2. Common audit lifecycle

Per D13 (Audit to Remediate):
- Plan → Execute → Findings → Auditee Response → CAPA → Effectiveness → Close

Each finding becomes a CAPA in HESEM (D6). Effectiveness verified
before finding closes.

---

## 3. Audit pack export

When inspector / auditor schedules audit, HESEM produces a signed
audit pack within 24-hour SLA. Per regulator + per tenant:
- Validation Master Plan + IQ/OQ/PQ records
- Audit trails for sampled records
- Change control history
- Training records
- Risk assessment
- Incident log
- Periodic review records
- Access control list
- Backup / DR records
- Penetration test report
- Per-vertical-pack-specific contents (Pharma APR, Auto PPAP, Aero AS9102)

---

## 4. Calendar coordination

Multi-audit calendar maintained per tenant. Coordination prevents
audit-fatigue (back-to-back audits with no preparation time).
