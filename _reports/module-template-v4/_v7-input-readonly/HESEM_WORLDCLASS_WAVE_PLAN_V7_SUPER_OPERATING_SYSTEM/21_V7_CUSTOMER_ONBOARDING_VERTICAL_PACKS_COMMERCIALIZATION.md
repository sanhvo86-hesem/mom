# 21 — Customer Onboarding, Vertical Packs and Commercialization
## Commercial principle

HESEM should not sell a generic “ERP/MES/QMS”. It should package vertical operating models: industrial manufacturing, pharma, medical device, automotive, aerospace, each with roots, validations, templates and onboarding playbooks.

## Vertical packs

| Vertical | Core differentiators | Mandatory roots |
| --- | --- | --- |
| Industrial manufacturing | dispatch, WIP, OEE, NC/CAPA, maintenance | SO/JO/WO/DISP/INSP/NQCASE/CAPA/MWO/OEEVT |
| Pharma | GMP, eBR, batch release, Part 11/Annex 11 | MREC/EBR/BREL/CDOC/TRAIN/ESIGN/AUDIT |
| Medical device | eDHR, DMR, complaint, traceability | EDHR/SERIAL/COMPLAINT/CAPA/CDOC/TRAIN |
| Automotive | APQP/PPAP/FMEA/control plan/supplier quality | APQP/PPAP/FMEA/CTRLPLAN/SCAR |
| Aerospace | FAI, serialization, concession, export control | FAI/SERIAL/CONC/EXPORT/ECO/IREV |

## Onboarding pack

- Site/tenant setup.
- Master data migration contract.
- Role/policy model.
- Process fit-gap.
- Validation/intended-use scoping if regulated.
- Training plan.
- Pilot runbook.
- Support/SRE and escalation.
