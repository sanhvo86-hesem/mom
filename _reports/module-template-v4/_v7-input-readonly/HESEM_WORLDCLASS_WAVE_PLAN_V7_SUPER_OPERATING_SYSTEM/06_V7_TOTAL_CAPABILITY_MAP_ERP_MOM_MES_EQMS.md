## Capability decomposition rule

Mỗi capability phải được chia thành: root contract, route contract, screen contract, workflow contract, API contract, event contract, evidence contract, data contract, security control, E2E test, rollback procedure, report output. Nếu không tách được như vậy, capability vẫn chỉ là idea, chưa là implementation-ready backlog.
