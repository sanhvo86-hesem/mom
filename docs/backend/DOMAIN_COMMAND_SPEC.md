# Domain Command Specification

This file defines the mandatory command layer that must replace direct business mutation through Generic CRUD or fragmented JSON services. Every command below is a backend API contract, not frontend behavior. Frontend must call process commands only after these APIs exist and are marked runtime-safe.

## Common Command Contract

All commands must implement the same envelope:

```http
POST /api/v1/commands/{CommandName}
Idempotency-Key: <required-unique-key>
X-Correlation-Id: <required-correlation-id>
X-CSRF-Token: <required-for-browser-session>
```

Common rules:

- The command starts one PostgreSQL transaction and commits only after all state, ledger, audit, evidence, and outbox writes are durable.
- Idempotency is checked before domain work. Same key and same fingerprint returns the same response. Same key and different fingerprint returns `409 idempotency_conflict`.
- All governed commands call `CreateElectronicSignature` or validate an existing signature when the command is regulated, financial, shipment-release, quality-disposition, or override-related.
- All commands write `command_audit`, domain-specific audit trail, and outbox event rows.
- Every error response includes `error.code`, `error.message`, `correlation_id`, `command`, `retryable`, and `violations[]`.
- Generic CRUD may not call these command internals.

## Additional Commands Required By Simulation

The following commands are not in the initial mandatory list but are required to close end-to-end logic:

| Command | Why required | Acceptance criterion |
| --- | --- | --- |
| `SplitLot` | Traceability simulation includes lot split. | Child lots inherit parent genealogy, cost, quality status, and hold state. |
| `MergeLot` | Traceability simulation includes merge. | Merged lot records all parent lots and blocks merge if any parent is held unless approved. |
| `ProduceOutputLotSerial` | MES completion must create output lot/serial. | Output lot/serial links WO, input lots, operations, inspections, and DPP. |
| `GenerateDppFromShipment` | DPP must not be manually decorative. | Shipment creates or updates DPP passport from genealogy and evidence. |
| `SimulateRecall` | Recall proof required. | Given lot/serial/customer/date, returns forward/backward chain with affected shipments and evidence. |
| `RecordMachineEvent` | Machine/MES simulation needs raw immutable event ingestion. | Stores raw event with source timestamp, quality code, adapter identity, replay key. |
| `DeriveProductionEvent` | Machine event must drive MES state. | Raw stream derives cycle/downtime/alarm/OEE events idempotently. |
| `PostPayment` | P2P simulation ends with payment. | Payment allowed only for matched/approved AP invoice and open period. |
| `ApplyQualityHold` / `ReleaseQualityHold` | Holds are currently inconsistent. | Hold store/table is canonical and all gates query it. |
| `PostCopqEntry` | COPQ JSONL is not canonical. | COPQ ledger posts from NCR/MRB/rework/scrap/warranty with period control. |
| `CreateQuote`, `SubmitQuoteForInternalReview`, `SendQuote`, `AcceptQuote` | Quote lifecycle needs the same idempotent command discipline as conversion. | Quote create/review/send/accept cannot bypass quote workflow or audit. |
| `RecordContractReview` | Current contract review is stored separately from SO and shipment gate reads a different shape. | SO confirmation and shipment gate read the same canonical review record. |
| `CreatePurchaseOrder`, `ApprovePurchaseOrder` | P2P simulation starts before receiving; PO creation/approval cannot stay generic CRUD. | Receiving command accepts only approved/open PO and ASL-approved supplier. |
| `PostArInvoice`, `CloseSalesOrder` | Quote-to-cash simulation includes invoice and SO close. | SO close requires shipment, invoice, no open quality/finance exceptions, and period policy. |
| `OpenFinancePeriod` | Finance control simulation begins with open period. | Posting commands can verify period state from the same period authority. |

## Compatibility And Rollback

- During migration, legacy JSON APIs may remain read-only behind feature flag `legacy_runtime_read_enabled`.
- Command services must dual-write JSON only in `SHADOW_WRITE` mode until reconciliation passes.
- Rollback from `POSTGRES_PRIMARY` to JSON is allowed only if drift report has zero missing rows for governed collections and no command was accepted after the rollback snapshot time.
- Existing endpoints remain but return `410 gone` or `403 forbidden` for governed mutations once command API is live.
